JavaScript & TypeScript Tutorial for Beginners

Welcome! This tutorial starts from absolute first principles: what JavaScript is, what Node.js is, and why we can run code "outside the browser". You'll learn JavaScript in a way that prepares you to write clean, component-style code, then transition smoothly into TypeScript.

No web development experience required. No HTML or CSS needed. Just pure programming fundamentals.

Part 1: JavaScript Fundamentals

1. What is JavaScript?

JavaScript (JS) is a programming language that lets you write instructions for a computer to execute. Originally designed for web browsers, JavaScript has evolved into a versatile language used in many contexts.

You can use JavaScript in two main environments:

  1. In a web browser (Chrome, Safari, Firefox, etc.) - for interactive websites
  2. Outside the browser using Node.js - for servers, tools, and applications

What is Node.js?

Node.js is a runtime environment - a program you install on your computer that can execute JavaScript files directly, just like Python or Java.

Key Understanding:

  • JavaScript = The programming language (the "what")
  • Node.js = The tool that runs JavaScript on your computer (the "how")
  • Without Node: Your computer doesn't understand .js files
  • With Node: You can run .js files like any other program

2. Your First "Hello World" Program

Let's write and run the simplest possible JavaScript program.

Step A: Create a folder named js-basics on your computer

Step B: Inside that folder, create a file named hello.js

Step C: Put this code inside hello.js:

console.log("Hello world!");

Step D: Open a terminal in that folder and run:

node hello.js

You should see:

Hello world!
What just happened?
  • hello.js is a text file containing JavaScript code
  • node hello.js means "run this file using Node.js"
  • console.log(...) prints text to the terminal/console

4. Variables: Storing Data for Later Use

A variable is like a labeled container that holds a value. Think of it as a named box where you can store information and retrieve it later.

Using let (for values that change)

let score = 0;        // Initial value
score = 10;           // Changed value
score = score + 5;    // Now score is 15
console.log(score);   // Prints: 15

Using const (for values that stay the same)

const username = "Sam";
console.log(username);  // Prints: Sam

// username = "John";   // ❌ ERROR! Cannot change a const
Beginner Best Practice: Always use const by default. Only use let when you genuinely need to change the value later. This prevents accidental changes and makes your code easier to understand.

5. Strings: Working with Text

Strings are sequences of characters used to represent text. JavaScript offers multiple ways to work with strings.

String Concatenation

const first = "Ada";
const last = "Lovelace";

// Old style: using + operator
console.log(first + " " + last);  // Ada Lovelace

// Modern style: template literals (template strings)
console.log(`${first} ${last}`);  // Ada Lovelace
Template literals (backticks `) are the modern, preferred way. They're easier to read and allow you to embed expressions directly using ${...} syntax.

More String Examples

const age = 25;
const message = `I am ${age} years old`;
console.log(message);  // I am 25 years old

const calculation = `5 + 3 = ${5 + 3}`;
console.log(calculation);  // 5 + 3 = 8

6. Numbers and Arithmetic Operators

JavaScript provides standard mathematical operators for working with numbers.

const a = 10;
const b = 3;

console.log(a + b);  // 13  (addition)
console.log(a - b);  // 7   (subtraction)
console.log(a * b);  // 30  (multiplication)
console.log(a / b);  // 3.333... (division)
console.log(a % b);  // 1   (remainder/modulo)
console.log(a ** b); // 1000 (exponentiation: 10³)

More Complex Operations

// Order of operations follows standard math rules (PEMDAS)
const result = (10 + 5) * 2;  // 30

// Increment and decrement
let counter = 0;
counter++;  // counter is now 1
counter--;  // counter is now 0 again

// Compound assignment
let total = 10;
total += 5;  // Same as: total = total + 5
total *= 2;  // Same as: total = total * 2

7. Making Decisions: If Statements

Programs need to make choices based on conditions. The if statement lets your code take different paths depending on whether something is true or false.

Basic If-Else

const age = 18;

if (age >= 18) {
  console.log("Adult");
} else {
  console.log("Not adult");
}

Multiple Conditions

const score = 85;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 80) {
  console.log("Grade: B");
} else if (score >= 70) {
  console.log("Grade: C");
} else {
  console.log("Grade: F");
}

8. Repeating Work: Loops

Loops allow you to execute code multiple times without writing the same code over and over.

The For Loop

Use when you know exactly how many times to repeat:

// Count from 1 to 5
for (let i = 1; i <= 5; i++) {
  console.log("Count:", i);
}
// Output: Count: 1, Count: 2, Count: 3, Count: 4, Count: 5
For Loop Anatomy:
  • let i = 1 - Initialize counter
  • i <= 5 - Continue while this is true
  • i++ - Increment counter after each iteration

The For...Of Loop (Easiest for Arrays)

Use when you want to loop through each item in a list:

const names = ["Ava", "Noah", "Mia"];

for (const name of names) {
  console.log("Hello,", name);
}
// Output: Hello, Ava
//         Hello, Noah
//         Hello, Mia

While Loop

Use when you don't know how many times you'll need to repeat:

let count = 0;

while (count < 3) {
  console.log("Count is:", count);
  count++;
}

9. Functions: Reusable Blocks of Logic

A function is a named set of instructions that you can run (call) whenever needed. Functions are the foundation of component-style programming.

Basic Function

// Define the function
function add(x, y) {
  return x + y;
}

// Use (call) the function
const result = add(5, 7);
console.log(result);  // 12

Functions with Multiple Statements

function greet(name) {
  const message = `Hello, ${name}!`;
  console.log(message);
  return message;  // Can also return the value
}

greet("Alice");  // Prints: Hello, Alice!

Arrow Functions (Modern Syntax)

// Traditional function
function multiply(a, b) {
  return a * b;
}

// Arrow function (shorter syntax)
const multiply = (a, b) => {
  return a * b;
};

// Even shorter (when only returning one expression)
const multiply = (a, b) => a * b;

Why Functions Matter for Components:

A component is essentially:

  • Inputs - Data passed to the function (parameters)
  • Processing - Logic inside the function
  • Outputs - Data returned from the function

Mastering functions is Step #1 toward component-style programming!

10. Objects: Grouping Related Data

Objects are the #1 data structure you'll use for component inputs. They group related pieces of data together with named properties.

Creating and Using Objects

const settings = {
  model: "example-model",
  temperature: 0.7,
  maxTokens: 200
};

// Accessing object properties
console.log(settings.model);        // example-model
console.log(settings.temperature);  // 0.7
console.log(settings.maxTokens);    // 200

Destructuring (Modern and Clean)

Extract multiple properties at once:

const { model, maxTokens } = settings;
console.log(model);     // example-model
console.log(maxTokens); // 200

// This is the same as:
const model = settings.model;
const maxTokens = settings.maxTokens;

Nested Objects

const user = {
  name: "Alice",
  age: 30,
  address: {
    city: "New York",
    country: "USA"
  }
};

console.log(user.address.city);  // New York
Think of destructuring as "unpacking" a config object. This pattern is everywhere in modern JavaScript and component-style code.

11. Arrays: Lists of Items

Arrays store ordered lists of items - perfect for messages, inputs, results, or any collection of data.

Creating and Modifying Arrays

const items = ["a", "b", "c"];

// Add to end
items.push("d");
console.log(items);  // ["a", "b", "c", "d"]

// Remove from end
const last = items.pop();
console.log(last);   // "d"

// Access by index (starts at 0)
console.log(items[0]);  // "a"
console.log(items[1]);  // "b"

Essential Array Methods

const numbers = [1, 2, 3, 4, 5];

// map: transform each item
const doubled = numbers.map(n => n * 2);
console.log(doubled);  // [2, 4, 6, 8, 10]

// filter: keep items that match a condition
const evens = numbers.filter(n => n % 2 === 0);
console.log(evens);  // [2, 4]

// find: get first item that matches
const firstBig = numbers.find(n => n > 3);
console.log(firstBig);  // 4

// reduce: combine all items into one value
const sum = numbers.reduce((total, n) => total + n, 0);
console.log(sum);  // 15

Looping Through Arrays

const fruits = ["apple", "banana", "cherry"];

// Method 1: for...of (cleanest)
for (const fruit of fruits) {
  console.log(fruit);
}

// Method 2: forEach (functional style)
fruits.forEach(fruit => {
  console.log(fruit);
});

// Method 3: traditional for loop
for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}

12. Asynchronous Programming: Waiting for Work to Complete

This is the most important modern JavaScript concept. In real applications, many operations take time:

  • Reading files from disk
  • Making network requests (API calls)
  • Querying databases
  • Any operation that requires waiting

Synchronous vs Asynchronous

  • Synchronous (sync): Code runs line by line, each line waits for the previous to finish
  • Asynchronous (async): You start an operation, let it run in the background, then handle the result when it's ready

The Modern Way: async/await

// Helper function that returns a Promise
function wait(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

// Async function can use 'await' to wait for Promises
async function run() {
  console.log("Start");
  await wait(1000);  // Wait 1 second
  console.log("After 1 second");
  await wait(1000);  // Wait another second
  console.log("After 2 seconds");
}

run();

Real-World Example: Fetching Data

// Simulated API call
async function fetchUserData(userId) {
  console.log("Fetching user data...");
  await wait(1000);  // Simulate network delay
  
  return {
    id: userId,
    name: "Alice",
    email: "alice@example.com"
  };
}

async function main() {
  const user = await fetchUserData(123);
  console.log("User:", user);
  console.log("Email:", user.email);
}

main();
This skill is essential for component-style code. Almost all real-world components perform asynchronous operations.

13. Error Handling: Protecting Your Code

Good code anticipates problems and handles them gracefully. The try/catch block lets you catch and handle errors.

Basic Try/Catch

function parseJson(text) {
  return JSON.parse(text);
}

try {
  const data = parseJson("{ bad json }");
  console.log(data);
} catch (err) {
  console.log("Error parsing JSON:", err.message);
}

With Async Functions

async function fetchData(url) {
  try {
    // Simulate API call that might fail
    if (url === "") {
      throw new Error("URL cannot be empty");
    }
    
    const data = await someAsyncOperation(url);
    return data;
  } catch (err) {
    console.error("Failed to fetch:", err.message);
    return null;  // Return safe default
  }
}

Component-Style Error Handling:

  • Validate inputs early and clearly
  • Fail with helpful messages that explain what went wrong
  • Use try/catch around risky operations
  • Return safe defaults when appropriate

14. Modules: Organizing Code Across Files

Real projects split code into multiple files. Modules let you export code from one file and import it into another.

Example: Math Utilities

File: math.js

// Export individual functions
function add(a, b) {
  return a + b;
}

function subtract(a, b) {
  return a - b;
}

function multiply(a, b) {
  return a * b;
}

// Export them
module.exports = { add, subtract, multiply };

File: index.js

// Import from math.js
const { add, multiply } = require("./math");

console.log(add(2, 3));        // 5
console.log(multiply(4, 5));   // 20

Run it:

node index.js

Modern ES Modules (Alternative Syntax)

File: math.mjs

// Modern export syntax
export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}

File: index.mjs

// Modern import syntax
import { add, multiply } from "./math.mjs";

console.log(add(2, 3));
console.log(multiply(4, 5));
Why This Matters: Component-style programming is all about creating reusable, modular code. Each component typically lives in its own file and exports its functionality.

Part 2: TypeScript - JavaScript with Type Safety

What is TypeScript?

TypeScript is JavaScript plus type annotations. It helps you catch mistakes before running your code.

The Problem TypeScript Solves:

// JavaScript allows this mistake:
const x = "10";         // x is a string
console.log(x * 2);     // 20 (JavaScript converts it)
// But what if you didn't intend that?

// TypeScript catches it:
const x: number = "10";  // ❌ ERROR: Type 'string' not assignable to 'number'

1. Installing TypeScript

In your project folder, run these commands:

# Initialize a new Node.js project
npm init -y

# Install TypeScript as a development dependency
npm install -D typescript

# Create TypeScript configuration file
npx tsc --init
This creates a tsconfig.json file that configures how TypeScript compiles your code.

2. Your First TypeScript File

Create a file named index.ts:

const name: string = "Sam";
const age: number = 30;
const isActive: boolean = true;

console.log(name, age, isActive);

Compile TypeScript to JavaScript:

npx tsc

This creates index.js. Now run it:

node index.js

The TypeScript Workflow:

  1. Write code in .ts files (TypeScript)
  2. Compile with npx tsc → creates .js files (JavaScript)
  3. Run the JavaScript with node

TypeScript lets you specify types for function parameters and return values.

Basic Typed Function

5. Component-Style Function Pattern

This is the pattern you'll use constantly for building components. It combines everything you've learned.

// Define input shape
type Input = {
  text: string;
  repeat: number;
};

// Define output shape
type Output = {
  result: string;
  length: number;
};

// Validation function
function validateInput(input: Input): void {
  if (input.repeat < 1) {
    throw new Error("repeat must be at least 1");
  }
  if (input.repeat > 100) {
    throw new Error("repeat cannot exceed 100");
  }
  if (!input.text || input.text.trim() === "") {
    throw new Error("text cannot be empty");
  }
}

// Main processing function
export async function run(input: Input): Promise<Output> {
  // Step 1: Validate
  validateInput(input);
  
  // Step 2: Process
  const repeated = Array(input.repeat).fill(input.text).join(" ");
  
  // Step 3: Return structured output
  return {
    result: repeated,
    length: repeated.length
  };
}

// Example usage
async function example() {
  try {
    const output = await run({
      text: "Hello",
      repeat: 3
    });
    
    console.log(output.result);  // "Hello Hello Hello"
    console.log(output.length);  // 17
  } catch (err) {
    console.error("Error:", err.message);
  }
}

example();

This Pattern Teaches:

  • ✅ Clean inputs - Well-defined type for what goes in
  • ✅ Validation - Check inputs before processing
  • ✅ Predictable output - Well-defined type for what comes out
  • ✅ Async-ready - Can handle asynchronous operations
  • ✅ Error handling - Fails gracefully with clear messages
  • ✅ Exportable - Can be imported and reused in other files

Advanced TypeScript Features

Union Types (Multiple Possible Types)

type Status = "pending" | "success" | "error";

function handleStatus(status: Status) {
  if (status === "pending") {
    console.log("Loading...");
  } else if (status === "success") {
    console.log("Done!");
  } else {
    console.log("Failed!");

Example: Complete Component

Here's a complete, real-world style component combining all concepts:

// types.ts - Type definitions
export type UserInput = {
  username: string;
  email: string;
  age?: number;
};

export type ValidationResult = {
  isValid: boolean;
  errors: string[];
};

export type ProcessedUser = {
  username: string;
  email: string;
  age: number;
  createdAt: string;
};

// validators.ts - Validation logic
export function validateUser(input: UserInput): ValidationResult {
  const errors: string[] = [];
  
  if (!input.username || input.username.length < 3) {
    errors.push("Username must be at least 3 characters");
  }
  
  if (!input.email || !input.email.includes("@")) {
    errors.push("Valid email is required");
  }
  
  if (input.age !== undefined && (input.age < 0 || input.age > 120)) {
    errors.push("Age must be between 0 and 120");
  }
  
  return {
    isValid: errors.length === 0,
    errors
  };
}

// component.ts - Main component logic
import { UserInput, ValidationResult, ProcessedUser } from "./types";
import { validateUser } from "./validators";

export async function processUser(input: UserInput): Promise<ProcessedUser> {
  // Validate input
  const validation = validateUser(input);
  
  if (!validation.isValid) {
    throw new Error(`Validation failed: ${validation.errors.join(", ")}`);
  }
  
  // Simulate async processing (e.g., database save)
  await new Promise(resolve => setTimeout(resolve, 100));
  
  // Return processed result
  return {
    username: input.username,
    email: input.email.toLowerCase(),
    age: input.age ?? 0,  // Default to 0 if not provided
    createdAt: new Date().toISOString()
  };
}

// index.ts - Usage example
import { processUser } from "./component";

async function main() {
  try {
    const result = await processUser({
      username: "alice123",
      email: "Alice@Example.com",
      age: 25
    });
    
    console.log("User processed:", result);
  } catch (err) {
    console.error("Failed to process user:", err.message);
  }
}

main();
