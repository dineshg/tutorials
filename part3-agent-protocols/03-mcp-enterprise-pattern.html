Updated Example: FastAPI → MCP Server → Salesforce

With Authentication and MCP Primitives: Tools / Resources / Prompts

Three Separate Auth Boundaries in Your Bank-Style Setup

1. User → FastAPI (your app auth)

User logs in with Google Workspace (OIDC). React calls FastAPI with Authorization: Bearer <user_access_token>.

2. FastAPI → MCP server (service-to-service auth) ✅ THIS IS WHAT WE ADD HERE

Your FastAPI backend calls the MCP server using a separate internal Bearer token (or OAuth/JWT/mTLS in real prod). This prevents "anyone who can reach the MCP URL" from using it.

3. MCP server → Salesforce (downstream service auth)

MCP server authenticates to Salesforce as an integration identity (service-level) using OAuth JWT Bearer flow.

Part A — MCP Salesforce Server

Components: Auth Middleware + Resource + Prompt + Tool

FastMCP Features Used:

  • Middleware: Intercept requests for authentication
  • Resources: Read-only URIs for policies/schemas/guardrails
  • Prompts: Server-defined templates to guide clients/agents consistently
  • Tools: Executable actions that interact with external systems

mcp_salesforce_server.py
# pip install fastmcp httpx PyJWT cryptography cachetools

import os
import re
import time
from typing import Any, Dict, Optional, TypedDict

import httpx
import jwt
from cachetools import TTLCache

from fastmcp import FastMCP
from fastmcp.exceptions import ToolError
from fastmcp.server.dependencies import get_http_headers
from fastmcp.server.middleware import Middleware, MiddlewareContext

mcp = FastMCP("DocQA-Salesforce-MCP", json_response=True)

# -----------------------------------------------------------------------------
# 1) SERVICE-TO-SERVICE AUTH (FastAPI -> MCP)
#    MCP server requires: Authorization: Bearer 
# -----------------------------------------------------------------------------
MCP_SERVICE_TOKEN = os.environ["MCP_SERVICE_TOKEN"]

class ServiceAuthMiddleware(Middleware):
    async def on_request(self, context: MiddlewareContext, call_next):
        # Runs for MCP "request" messages (list tools, call tool, read resource, get prompt, etc.)
        headers = get_http_headers()
        auth = headers.get("authorization") or headers.get("Authorization")

        if not auth or not auth.startswith("Bearer "):
            raise ToolError("Unauthorized: missing Bearer token to call MCP server")

        token = auth.removeprefix("Bearer ").strip()
        if token != MCP_SERVICE_TOKEN:
            raise ToolError("Unauthorized: invalid MCP service token")

        return await call_next(context)

mcp.add_middleware(ServiceAuthMiddleware())  # FastMCP middleware registration

# -----------------------------------------------------------------------------
# 2) SALESFORCE SERVICE-LEVEL AUTH (MCP -> Salesforce)
#    OAuth JWT Bearer flow to get access_token.
# -----------------------------------------------------------------------------
SF_LOGIN_URL   = os.environ.get("SF_LOGIN_URL", "https://login.salesforce.com").rstrip("/")
SF_CLIENT_ID   = os.environ["SF_CLIENT_ID"]           # Connected App consumer key
SF_USERNAME    = os.environ["SF_USERNAME"]            # Integration user username
SF_PRIVATE_KEY = os.environ["SF_PRIVATE_KEY_PEM"]     # PEM private key
SF_API_VERSION = os.environ.get("SF_API_VERSION", "v60.0")

_token_cache: TTLCache[str, Dict[str, Any]] = TTLCache(maxsize=1, ttl=14 * 60)  # ~14 min cache

def _build_jwt_assertion() -> str:
    now = int(time.time())
    payload = {
        "iss": SF_CLIENT_ID,
        "sub": SF_USERNAME,
        "aud": SF_LOGIN_URL,
        "exp": now + 180,  # short-lived assertion
    }
    return jwt.encode(payload, SF_PRIVATE_KEY, algorithm="RS256")

async def _get_salesforce_token() -> tuple[str, str]:
    cached = _token_cache.get("sf_token")
    if cached:
        return cached["access_token"], cached["instance_url"]

    token_url = f"{SF_LOGIN_URL}/services/oauth2/token"
    data = {
        "grant_type": "urn:ietf:params:oauth:grant-type:jwt-bearer",
        "assertion": _build_jwt_assertion(),
    }

    async with httpx.AsyncClient(timeout=10) as client:
        r = await client.post(token_url, data=data)
        if r.status_code != 200:
            raise RuntimeError(f"Salesforce token error: {r.status_code} {r.text}")
        tok = r.json()

    _token_cache["sf_token"] = {
        "access_token": tok["access_token"],
        "instance_url": tok["instance_url"],
    }
    return tok["access_token"], tok["instance_url"]

# -----------------------------------------------------------------------------
# 3) RESOURCE: publish a "bank-safe SOQL policy" as read-only context
# -----------------------------------------------------------------------------
@mcp.resource("resource://salesforce/soql_policy")
def soql_policy() -> str:
    return (
        "BANK POLICY:\n"
        "- Only SELECT queries\n"
        "- LIMIT required (<= 50)\n"
        "- No write keywords (INSERT/UPDATE/DELETE/UPSERT/MERGE)\n"
        "- Prefer allowlisting objects/fields in production\n"
    )

# -----------------------------------------------------------------------------
# 4) PROMPT: a reusable instruction template clients/agents can fetch
# -----------------------------------------------------------------------------
@mcp.prompt
def salesforce_safe_query_prompt() -> str:
    return (
        "You are generating Salesforce SOQL for a banking environment.\n"
        "Follow the SOQL policy from resource://salesforce/soql_policy.\n"
        "Always include LIMIT <= 50. Only SELECT.\n"
    )

# -----------------------------------------------------------------------------
# 5) TOOL: execute SOQL (after validation) via Salesforce Query REST endpoint
#    /services/data/vXX.X/query?q=
# -----------------------------------------------------------------------------
def _validate_soql_bank_safe(soql: str) -> None:
    s = soql.strip()
    if not re.match(r"(?is)^select\s", s):
        raise ValueError("Only SELECT queries are allowed")

    if re.search(r"(?is)\b(insert|update|delete|upsert|merge|drop)\b", s):
        raise ValueError("Write operations are not allowed")

    m = re.search(r"(?is)\blimit\s+(\d+)\b", s)
    if not m:
        raise ValueError("LIMIT is required")
    if int(m.group(1)) > 50:
        raise ValueError("LIMIT too high (max 50)")

async def _sf_query(instance_url: str, access_token: str, soql: str) -> Dict[str, Any]:
    url = f"{instance_url}/services/data/{SF_API_VERSION}/query"
    headers = {"Authorization": f"Bearer {access_token}"}

    async with httpx.AsyncClient(timeout=15) as client:
        r = await client.get(url, headers=headers, params={"q": soql})
        if r.status_code == 401:
            # token expired/invalid -> clear cache and retry once
            _token_cache.pop("sf_token", None)
            access_token, instance_url = await _get_salesforce_token()
            headers["Authorization"] = f"Bearer {access_token}"
            r = await client.get(url, headers=headers, params={"q": soql})

        if r.status_code != 200:
            raise RuntimeError(f"Query error: {r.status_code} {r.text}")

        return r.json()

@mcp.tool()
async def sf_execute_soql(soql: str) -> Dict[str, Any]:
    """
    Executes SOQL under the integration user's permissions.
    """
    _validate_soql_bank_safe(soql)
    access_token, instance_url = await _get_salesforce_token()
    return await _sf_query(instance_url, access_token, soql)

if __name__ == "__main__":
    # Streamable HTTP is the common production transport.
    mcp.run(transport="streamable-http")

Part B — FastAPI Host Endpoint

Calls MCP with Service Authentication

Key Feature:

FastMCP's Streamable HTTP client transport supports custom headers (or a BearerAuth helper) for service-to-service authentication.

Save as: main.py

main.py
# pip install fastapi uvicorn fastmcp pydantic

import os
from fastapi import FastAPI, Depends, HTTPException
from pydantic import BaseModel

from fastmcp import Client
from fastmcp.client.transports import StreamableHttpTransport

app = FastAPI()

MCP_SF_URL = os.environ["MCP_SF_URL"]                  # e.g. http://mcp-salesforce.internal/mcp
MCP_SERVICE_TOKEN = os.environ["MCP_SERVICE_TOKEN"]    # same shared secret used by MCP server

class SoqlReq(BaseModel):
    soql: str

async def get_identity():
    # Placeholder: in your real app, verify Google OIDC Bearer token and return claims.
    return {"email": "alice@bank.com", "is_admin": True}

@app.post("/admin/salesforce/query")
async def admin_salesforce_query(req: SoqlReq, ident=Depends(get_identity)):
    # Bank rule: only admins can submit "free-form query text"
    if not ident.get("is_admin"):
        raise HTTPException(status_code=403, detail="Admin only")

    # Service-to-service auth to MCP server (NOT the end-user token)
    transport = StreamableHttpTransport(
        url=MCP_SF_URL,
        headers={"Authorization": f"Bearer {MCP_SERVICE_TOKEN}"},
    )
    mcp_client = Client(transport)

    async with mcp_client:
        # Optional: read policy/prompt for debugging or to feed an agent
        policy = await mcp_client.read_resource("resource://salesforce/soql_policy")
        prompt = await mcp_client.get_prompt("salesforce_safe_query_prompt")

        # Actual tool call
        res = await mcp_client.call_tool("sf_execute_soql", {"soql": req.soql})
        return {
            "policy": policy.content,   # what the server says is allowed
            "prompt": prompt.messages,  # how the server wants queries generated
            "result": res,
        }

What Changed: The "Auth Story"

Key Security Improvements

  • FastAPI doesn't just "call the MCP URL." It must present an MCP service token (or OAuth/JWT/mTLS in stricter setups).
  • The MCP server enforces that token in middleware before allowing any MCP operation (tools/resources/prompts/listing).
  • Only after that does the MCP tool do Salesforce service-level auth (JWT Bearer) and run SOQL via Salesforce's Query REST endpoint.

Authentication Flow

User (Google OIDC)

↓

FastAPI (User token validated)

↓

MCP Server (Service token validated in middleware)

↓

Salesforce (OAuth JWT Bearer flow)

Next Level Security:

If you want to further tighten the "bank-safe" part, the client should never send arbitrary SOQL. Instead, send only parameters (like email) while the MCP server constructs a fixed allowlisted query internally.

Environment Variables Required

MCP Server (mcp_salesforce_server.py):

  • MCP_SERVICE_TOKEN - Shared secret for FastAPI → MCP auth
  • SF_CLIENT_ID - Salesforce Connected App consumer key
  • SF_USERNAME - Integration user username
  • SF_PRIVATE_KEY_PEM - PEM private key for JWT signing
  • SF_LOGIN_URL - (Optional) Default: https://login.salesforce.com
  • SF_API_VERSION - (Optional) Default: v60.0

FastAPI Host (main.py):

  • MCP_SF_URL - MCP server URL (e.g., http://mcp-salesforce.internal/mcp)
  • MCP_SERVICE_TOKEN - Same shared secret as MCP server

Summary: Why This Architecture Matters

Defense in Depth:

  1. User Authentication: Validates end-user identity via Google OIDC
  2. Service Authentication: Prevents unauthorized access to MCP server
  3. Downstream Authentication: Secures integration with Salesforce
  4. Query Validation: Enforces bank-safe SOQL policies at MCP layer
  5. Resource Policies: Provides read-only context for LLM agents
  6. Prompt Templates: Standardizes agent behavior across clients

Result: A secure, auditable system where each layer has its own authentication boundary and the MCP server acts as a controlled gateway to sensitive operations.
