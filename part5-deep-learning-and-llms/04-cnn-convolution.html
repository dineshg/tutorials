1) Convolution as an image modifier

Convolution takes an input image and a small filter (also called a kernel) and produces an output image. We write this conceptually as:

\[ \text{output} = \text{input} \;\star\; \text{kernel} \]

In math you often see a star/asterisk for convolution. In code, convolution is not written with * because * already means multiplication.

Example A: blurring
Using a blur kernel (like Gaussian blur) produces a softened image.

Example B: edge detection
Using an edge kernel produces strong responses at boundaries (edges) and low responses elsewhere.

Why filters matter: the kernel determines what transformation happens. Change the kernel → you change the effect. CNNs learn these kernels automatically from data using gradient-based optimization.

2) Convolution mechanics: just multiply + add

Convolution “slides” the kernel over the image. At each position, it performs element-wise multiplication and sums the result. That single number becomes one pixel in the output.

Tiny worked example

We’ll use a 4×4 input and a 2×2 kernel. (Small on purpose so you can verify by hand.)

\[ 1\cdot 0 + 0\cdot 10 + 0\cdot 20 + 2\cdot 30 = 60 \]

\[ 1\cdot 10 + 0\cdot 10 + 0\cdot 30 + 2\cdot 30 = 70 \]

\[ 1\cdot 10 + 0\cdot 0 + 0\cdot 30 + 2\cdot 20 = 50 \]

3) Output size and convolution “modes” (Valid / Same / Full)

If the input has height \(H\), width \(W\), and the kernel has height \(K_H\), width \(K_W\), then with no padding and stride 1, the output size is:

\[ H_{\text{out}} = H - K_H + 1,\quad W_{\text{out}} = W - K_W + 1 \]

This is the “valid” mode: you only place the kernel where it fully fits inside the input.

Padding: making output the same size

If you pad the input with zeros around the border, you allow the kernel to slide beyond the original edges. With padding \(P_H, P_W\) and stride 1:

\[ H_{\text{out}} = H + 2P_H - K_H + 1,\quad W_{\text{out}} = W + 2P_W - K_W + 1 \]

To get same output size (i.e., \(H_{\text{out}}=H\), \(W_{\text{out}}=W\)) with odd square kernels (like \(3\times 3\), \(5\times 5\)), choose:

\[ P = \frac{K-1}{2} \]

Full mode: “catch everything”

If you keep padding so the kernel can slide over every partial overlap, the 1D output length becomes: \[ n_{\text{out}} = n + k - 1 \] (and similarly in 2D). This is the classic “full” convolution size.

Practical note: images “in the wild” are often rectangular, but kernels are usually square by convention (e.g., 3×3, 5×5).

4) Pseudocode and a direct implementation

A straightforward 2D convolution (valid, stride 1) can be written with four loops: two for output position \((i,j)\) and two for kernel position \((i',j')\).

python • naive 2D “deep learning style” convolution (cross-correlation), valid mode 
import numpy as np

def corr2d_valid(A, W):
    """
    Cross-correlation (deep-learning 'convolution') in 2D, valid mode, stride=1.
    A: (H, W) input
    W: (Kh, Kw) kernel
    returns: (H-Kh+1, W-Kw+1)
    """
    H, W_in = A.shape
    Kh, Kw = W.shape
    H_out = H - Kh + 1
    W_out = W_in - Kw + 1
    out = np.zeros((H_out, W_out), dtype=float)

    for i in range(H_out):
        for j in range(W_out):
            s = 0.0
            for ii in range(Kh):
                for jj in range(Kw):
                    s += A[i + ii, j + jj] * W[ii, jj]
            out[i, j] = s
    return out

A = np.array([[0,10,10,0],
              [20,30,30,20],
              [10,20,20,10],
              [0, 5, 5, 0]], dtype=float)

W = np.array([[1,0],
              [0,2]], dtype=float)

print(corr2d_valid(A, W))

This code matches the lecture’s “multiply + add” definition and produces the 3×3 output for the 4×4 input with a 2×2 kernel.

5) The defining equation (2D)

The 2D deep-learning version (cross-correlation) at output location \((i,j)\) is:

\[ (A \star W)_{i,j} = \sum_{i'=0}^{K_H-1}\sum_{j'=0}^{K_W-1} A_{i+i',\, j+j'} \, W_{i',\, j'} \]

This equation is exactly what the nested loops compute.

6) “Convolution” vs cross-correlation (and why deep learning uses the latter)

In classic signal processing, convolution flips the kernel (reverses its orientation). In deep learning libraries, the operation called “convolution” typically does not flip the kernel.

Cross-correlation (deep learning style):
slide the kernel as-is

Convolution (signal processing):
slide a flipped kernel

If the kernel is learned automatically (as in CNN training), the flip convention doesn’t change what the model can learn: the optimizer can learn flipped patterns just as easily.

Takeaway: it’s helpful to think of CNN kernels as patterns and the operation as a sliding pattern matcher.

7) Convolution as a sliding dot product (similarity / correlation view)

A dot product between vectors \(a\) and \(b\) can be written in two equivalent ways:

\[ a \cdot b = \sum_{i=1}^d a_i b_i \]

\[ a \cdot b = \|a\| \, \|b\| \cos(\theta) \]

The cosine term \(\cos(\theta)\) is a similarity measure:

  • \(\theta=0^\circ \Rightarrow \cos(\theta)=1\) (most similar)
  • \(\theta=90^\circ \Rightarrow \cos(\theta)=0\) (orthogonal / unrelated)
  • \(\theta=180^\circ \Rightarrow \cos(\theta)=-1\) (opposite)

Convolution/correlation is essentially the same dot-product idea, but repeated over all image locations:

  1. take a local patch of the image (same shape as kernel)
  2. compute dot product with the kernel
  3. store result in output at that location

Pattern-finder interpretation: the kernel encodes a pattern. Sliding it across the image asks: “how similar is this patch to my pattern?” High response → pattern present.

Cosine similarity vs Pearson correlation (conceptual link)

Pearson correlation looks like cosine similarity but with mean subtraction. That’s another reason the word “correlation” is a helpful mental model for what CNN filters do.

8) Convolution as matrix multiplication (1D “Toeplitz” view)

In 1D, a length-\(n\) input \(a=[a_1,a_2,\dots,a_n]\) convolved (in deep-learning style) with a length-\(k\) kernel \(w=[w_1,\dots,w_k]\) produces:

\[ b_i = \sum_{i'=1}^{k} a_{i+i'-1}\, w_{i'} \quad \text{for } i=1,\dots,n-k+1 \]

You can implement the same operation by building a matrix that repeats and shifts the kernel across rows (a Toeplitz-like matrix), then multiplying by the input vector. This shows convolution is a linear operation.

Why we don’t do this in practice: the equivalent matrix is huge because it repeats kernel parameters many times. CNNs avoid this by storing the small kernel once and reusing it everywhere (parameter sharing).

python • 1D convolution via explicit matrix multiplication (teaching only) 
import numpy as np

def conv1d_via_matrix(a, w):
    a = np.asarray(a, dtype=float)
    w = np.asarray(w, dtype=float)
    n = a.size
    k = w.size
    out_len = n - k + 1

    # Build "convolution matrix" M of shape (out_len, n)
    M = np.zeros((out_len, n), dtype=float)
    for i in range(out_len):
        M[i, i:i+k] = w  # place kernel in row i, shifted right by i

    b = M @ a
    return b, M

a = [1, 2, 3, 4]
w = [10, 100]
b, M = conv1d_via_matrix(a, w)

print("M=\n", M)
print("b=", b)

9) PyTorch conv2d quickstart (what CNN layers actually do)

A CNN layer applies many kernels to an input. In PyTorch:

  • input tensor shape: \((N, C_{\text{in}}, H, W)\)
  • output tensor shape: \((N, C_{\text{out}}, H_{\text{out}}, W_{\text{out}})\)
  • weights shape: \((C_{\text{out}}, C_{\text{in}}, K_H, K_W)\)

Important: this is still “multiply + add”, just extended across channels and many filters.

python • minimal conv2d example (demonstrates shapes, padding, valid vs same-like) 
import torch
import torch.nn as nn

# N=1 image, Cin=1 channel, H=W=4
x = torch.tensor([[[[0., 10., 10., 0.],
                    [20., 30., 30., 20.],
                    [10., 20., 20., 10.],
                    [0.,  5.,  5.,  0.]]]])

# One output channel, one input channel, kernel 2x2
conv = nn.Conv2d(in_channels=1, out_channels=1, kernel_size=2, bias=False)

# Manually set weights to match the earlier kernel [[1,0],[0,2]]
with torch.no_grad():
    conv.weight[:] = torch.tensor([[[[1., 0.],
                                     [0., 2.]]]])

y_valid = conv(x)              # padding=0 by default -> "valid"
print("valid out shape:", y_valid.shape)
print(y_valid[0,0])

# If you want "same" output size, you typically pad (for odd kernels).
# Here kernel is 2x2; "same" is less symmetric. We'll show padding=1 anyway:
conv_pad = nn.Conv2d(1, 1, kernel_size=2, padding=1, bias=False)
with torch.no_grad():
    conv_pad.weight[:] = conv.weight

y_padded = conv_pad(x)
print("padded out shape:", y_padded.shape)
print(y_padded[0,0])

This example uses PyTorch’s deep-learning “convolution” (cross-correlation). The values match the hand computation for valid mode.
