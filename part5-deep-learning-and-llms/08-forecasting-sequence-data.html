Forecasting & Sequence Data (Time Series)

A standalone tutorial on: what sequence data is, how to shape it for ML, what “forecasting” really means, and how to avoid the most common multi-step forecasting mistake (data leakage via future inputs).

Key idea

Sequences aren’t just “lots of data.” They have order. The value at time \(t\) is usually related to values at times \(t-1, t-2, \dots\). Forecasting is about exploiting that dependence to predict the future.

1) What counts as sequence data?

Time series (continuous-valued sequences)

  • Finance: prices / returns sampled every minute/day
  • Weather: temperature, rain, wind over time
  • Audio: a waveform is literally amplitude vs time
  • Sensor streams: accelerometers, ECG, industrial sensors

The “sequence” part is the time axis: data points arrive in order and nearby points tend to be related.

Example time series plot (online)

Text is also sequential (but not continuous-valued)

A sentence is a sequence of tokens. Historically, older ML often used bag-of-words: convert a document into a word-count vector. That makes it easy to use standard classifiers, but it destroys word order.

Why bag-of-words can fail: “dog toy” vs “toy dog” have the same word counts but different meaning.

2) How to represent sequence data: shape \(N \times T \times D\)

For non-sequential data you’re used to \(X \in \mathbb{R}^{N \times D}\) (samples × features). For sequences, we add a time axis:

\[ X \in \mathbb{R}^{N \times T \times D} \]

  • \(N\): number of samples (each sample is one whole sequence)
  • \(T\): number of time steps per sequence
  • \(D\): features measured at each time step

Concrete examples (to stop mixing up \(N\), \(T\), \(D\))

  • GPS commute tracking: each trip = one sample (\(N\)); each timestamp = one step (\(T\)); lat/long so \(D=2\).
  • Single stock price: \(D=1\). If you slice into windows of length \(T\), then each window is one sample.
  • 500 stocks: \(D=500\). Each time step contains a 500-dimensional snapshot.
  • Brain electrodes: \(D=\) #electrodes. With 1 kHz sampling for 1 second, \(T=1000\).

The axis order \(N \times T \times D\) is mostly convention in Python ML tooling.

3) What “forecasting” means (and what the horizon is)

In this course-style definition, forecasting means: given a time series, predict the next values. Usually we care about predicting multiple steps ahead.

Horizon \(H\): how far into the future you want predictions (in number of steps).

  • If steps are hours and you forecast 7 days hourly: \(H = 7 \times 24 = 168\).
  • If steps are days and you forecast next 5 days: \(H=5\).

4) A “dumb but great” baseline: Linear Regression as an AR model

Before you reach for RNNs, use a simple baseline. For a 1D time series (\(D=1\)), you can treat the previous \(p\) values as features and do linear regression.

\[ \hat{x}_t = w_0 + w_1 x_{t-1} + w_2 x_{t-2} + \dots + w_p x_{t-p} \]

In statistics this is an autoregressive (AR) model: predict a series using its own past.

Build training windows

If your full series has length \(L\) and you use a window length \(T\) to predict the next value, the number of input windows is:

\[ \text{windows} = L - (T+1) + 1 = L - T \]

Why \(T+1\)? Because you need \(T\) inputs plus 1 target step. 

import numpy as np

def make_supervised_1d(series: np.ndarray, lookback: int):
    """
    Converts a 1D time series into supervised learning data:
      X[i] = series[i : i+lookback]
      y[i] = series[i+lookback]
    """
    series = np.asarray(series, dtype=np.float32)
    L = len(series)
    if L <= lookback:
        raise ValueError("series is too short for the chosen lookback")

    X = []
    y = []
    for i in range(L - lookback):
        X.append(series[i : i + lookback])
        y.append(series[i + lookback])
    return np.stack(X), np.asarray(y)

# Example:
x = np.arange(10, dtype=np.float32)  # [0..9]
X, y = make_supervised_1d(x, lookback=3)
print(X.shape, y.shape)  # (7, 3) (7,)

5) The most common forecasting mistake: leaking future inputs

The wrong way (for multi-step forecasting):

Predicting day 5 using inputs that include the true day 4 value, when day 4 is not known at forecast time. This makes results look “nice” but it’s not a valid forecast.

The correct way (recursive / rolling forecast):

When you need multiple steps ahead, you must feed the model’s own previous predictions back as inputs. This matches reality: once you step into the future, you no longer have ground-truth observations.

6) Minimal, correct forecasting code (baseline AR with recursion)

Train a linear model (scikit-learn style) + recursive forecast

import numpy as np
from sklearn.linear_model import LinearRegression

def fit_ar_linear(train_series: np.ndarray, lookback: int) -> LinearRegression:
    X_train, y_train = make_supervised_1d(train_series, lookback)
    model = LinearRegression()
    model.fit(X_train, y_train)
    return model

def recursive_forecast_1d(model, history: np.ndarray, lookback: int, horizon: int):
    """
    history: last observed values (must be at least lookback long)
    Returns: horizon predictions [x̂(t), x̂(t+1), ...]
    """
    hist = list(np.asarray(history, dtype=np.float32))
    if len(hist) < lookback:
        raise ValueError("history must contain at least lookback values")

    preds = []
    for _ in range(horizon):
        x_in = np.array(hist[-lookback:], dtype=np.float32).reshape(1, -1)
        y_hat = float(model.predict(x_in)[0])
        preds.append(y_hat)
        hist.append(y_hat)  # <-- key step: feed prediction back in
    return np.array(preds, dtype=np.float32)

# --- Example usage (toy) ---
series = np.sin(np.linspace(0, 10*np.pi, 500)).astype(np.float32)
train, test = series[:400], series[400:]  # naive split for demo

lookback = 20
H = 30

ar = fit_ar_linear(train, lookback=lookback)
history = train[-lookback:]
preds = recursive_forecast_1d(ar, history, lookback=lookback, horizon=H)

print("First 5 preds:", preds[:5])

Important: this evaluates a true “forecast”: after the split point, you do not feed the model real future values.

7) Where RNNs fit (preview)

This tutorial intentionally starts with the simplest correct approach: an autoregressive baseline. RNNs (and later, modern sequence models) shine when relationships are nonlinear and depend on longer context, but the data-shape thinking (\(N \times T \times D\)) and the “no future leakage” rule stay the same.

If your baseline is wrong (e.g., uses true future inputs), you can’t trust any advanced model results either.

Checklist: a “correct” forecasting setup

  • Define horizon \(H\) (predict multiple steps, not just 1).
  • Split by time (train on past, test on future).
  • No leakage: during multi-step forecasting, do not use true future values as inputs.
  • Start with a baseline (AR/linear regression) before complex sequence models.
