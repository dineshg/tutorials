1) Mini-batches: the practical way to train on large datasets

Most training objectives are averages over a dataset. If your dataset has \(N\) samples and per-sample loss \(\ell_i(\theta)\), the empirical risk is:

\[ L(\theta) = \frac{1}{N}\sum_{i=1}^N \ell_i(\theta) \]

Computing \(\nabla_\theta L(\theta)\) over all \(N\) samples every step is expensive. Instead, we use a mini-batch \(B\) (a small random subset) and approximate the loss and gradient:

\[ \widehat{L}_B(\theta) = \frac{1}{|B|}\sum_{i\in B}\ell_i(\theta), \quad \nabla_\theta \widehat{L}_B(\theta) \approx \nabla_\theta L(\theta) \]

Intuition: if batches are shuffled, each batch is a “representative sample” of the dataset. Over many steps, you still see the whole dataset, but each update is much cheaper.

2) Data pipeline: datasets, transforms, and DataLoader

A clean PyTorch pipeline usually has three parts:

  1. Dataset: knows how to fetch one sample \((x,y)\)
  2. Transform(s): preprocessing applied per sample
  3. DataLoader: batching, shuffling, parallel loading

MNIST: why values become 0–1 only when transformed

MNIST is stored as 8-bit integers (0–255). transforms.ToTensor() converts to float and scales to \([0,1]\). If you inspect raw stored tensors, you’ll see 0–255. If you iterate through the DataLoader with the transform, you’ll see 0–1.

python • verify raw vs transformed MNIST ranges 
from torchvision import datasets, transforms

train_raw = datasets.MNIST(root="data", train=True, download=True, transform=None)
print("raw dtype:", train_raw.data.dtype)
print("raw min/max:", int(train_raw.data.min()), int(train_raw.data.max()))  # typically 0..255

train_tf = datasets.MNIST(root="data", train=True, download=True, transform=transforms.ToTensor())
x0, y0 = train_tf[0]
print("transformed dtype:", x0.dtype)
print("transformed min/max:", float(x0.min()), float(x0.max()))  # typically 0..1

Rule of thumb: keep preprocessing inside the Dataset via transform=.... That way every training/eval pass uses the same consistent inputs.

3) Training loop patterns that scale

A robust training loop separates:

  • training mode (updates parameters) vs evaluation mode (no updates)
  • metrics computed batch-by-batch
  • device transfers done once per batch

One update step on a batch

For a batch \((x_B, y_B)\), gradient descent performs:

\[ \theta \leftarrow \theta - \eta \nabla_\theta \widehat{L}_B(\theta) \]

Canonical PyTorch “inner loop”

python • canonical mini-batch update step 
model.train()                 # enables train-time behavior (e.g., dropout/bn)
for x, y in train_loader:
    x, y = x.to(device), y.to(device)

    optimizer.zero_grad()     # clear old gradients
    logits = model(x)         # forward pass
    loss = criterion(logits, y)

    loss.backward()           # compute gradients
    optimizer.step()          # update parameters

Epoch loss: average over batches

If you collect per-batch losses \(\{\widehat{L}_{B_k}\}\) in one epoch, a common summary is the mean: \[ \text{epoch\_loss} = \frac{1}{K}\sum_{k=1}^K \widehat{L}_{B_k} \] This is a practical proxy for “how training is going” without doing a second full pass just to re-measure loss.

4) Accuracy when your data arrives in batches

When data comes via a DataLoader, you usually can’t compute accuracy in one shot. The simple fix is to accumulate counts.

\[ \text{accuracy} = \frac{\sum_{i=1}^{N} \mathbb{1}\{\hat{y}_i = y_i\}}{N} \]

With batches, you compute “correct in this batch” and “batch size”, then add them up.

python • batched accuracy over a DataLoader 
@torch.no_grad()
def accuracy_loader(model, loader, device):
    model.eval()
    correct = 0
    total = 0
    for x, y in loader:
        x, y = x.to(device), y.to(device)
        logits = model(x)
        pred = logits.argmax(dim=1)           # class index
        correct += (pred == y).sum().item()
        total += y.numel()
    return correct / total

Tip: use @torch.no_grad() in evaluation to reduce memory usage and speed things up.

5) Logits vs probabilities: why argmax is enough for predictions

Many classification models output logits \(z \in \mathbb{R}^K\). Softmax converts logits into probabilities:

\[ \mathrm{softmax}(z)_i = \frac{e^{z_i}}{\sum_{j=1}^K e^{z_j}} \]

For choosing the predicted class, you almost never need the probabilities. You only need: \[ \hat{y} = \arg\max_i z_i \]

Key fact: argmax(softmax(z)) == argmax(z). Exponentials preserve ordering and the denominator is shared.

PyTorch: get indices (classes) from logits

python • logits → predicted class indices 
logits = model(x)            # shape: (B, K)
pred = torch.argmax(logits, dim=1)  # shape: (B,)

If you want probabilities for interpretability (e.g., confidence), you can still compute: probs = torch.softmax(logits, dim=1). For training with nn.CrossEntropyLoss, keep the model output as logits.

6) GPU usage: avoid “tensors on different devices” errors

In PyTorch, your model parameters live on a device (CPU or GPU). Your data must be on the same device before you do operations between them.

Do:
• model.to(device)
• x.to(device), y.to(device) each batch

Don’t:
• keep model on GPU and data on CPU (or vice versa)
• mix tensors across devices in one computation

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

7) Debugging classification: confusion matrix + misclassified samples

Overall accuracy hides which classes are being confused. A confusion matrix breaks errors down by class. For a \(K\)-class problem, it’s a \(K \times K\) table where entry \((i,j)\) counts: “true label \(i\) predicted as \(j\)”.

Collect predictions for a full DataLoader pass

python • gather all predictions + labels 
@torch.no_grad()
def collect_preds(model, loader, device):
    model.eval()
    all_pred = []
    all_y = []
    for x, y in loader:
        x = x.to(device)
        logits = model(x)
        pred = logits.argmax(dim=1).cpu()
        all_pred.append(pred)
        all_y.append(y.cpu())
    return torch.cat(all_y), torch.cat(all_pred)

Compute confusion matrix (two options)

Option A: pure PyTorch (no extra deps)

python • confusion matrix in pure torch 
def confusion_matrix_torch(y_true, y_pred, num_classes):
    cm = torch.zeros((num_classes, num_classes), dtype=torch.int64)
    for t, p in zip(y_true, y_pred):
        cm[t, p] += 1
    return cm

y_true, y_pred = collect_preds(model, test_loader, device)
cm = confusion_matrix_torch(y_true, y_pred, num_classes=10)
print(cm)

Option B: scikit-learn (convenient)

Find and visualize misclassified samples

Once you know common confusions, actually look at examples. (The code below generates plots when you run it.)

python • extract misclassified MNIST examples + plot a grid 
import matplotlib.pyplot as plt

@torch.no_grad()
def get_misclassified(model, loader, device, max_items=25):
    model.eval()
    items = []
    for x, y in loader:
        x, y = x.to(device), y.to(device)
        logits = model(x)
        pred = logits.argmax(dim=1)
        wrong = pred != y
        if wrong.any():
            xw = x[wrong].cpu()
            yw = y[wrong].cpu()
            pw = pred[wrong].cpu()
            for i in range(xw.size(0)):
                items.append((xw[i], int(yw[i]), int(pw[i])))
                if len(items) >= max_items:
                    return items
    return items

mis = get_misclassified(model, test_loader, device, max_items=16)

# Plot (runs locally; not generated in this HTML)
fig, axes = plt.subplots(4, 4, figsize=(6, 6))
for ax, (img, y, p) in zip(axes.ravel(), mis):
    ax.imshow(img.squeeze(0), cmap="gray")
    ax.set_title(f"true={y}, pred={p}", fontsize=9)
    ax.axis("off")
plt.tight_layout()
plt.show()

8) Synthetic regression: fit a non-linear surface you can visualize

Classification isn’t the only job for feed-forward nets. For regression, the output layer is typically identity (no activation), and the loss is often mean squared error (MSE).

Define a “bumpy” 2D function

We’ll sample \(x \in \mathbb{R}^2\) uniformly in a box and define targets:

\[ y = \cos(2x_1) + \cos(3x_2) \]

Generate data + 3D scatter (run locally)

python • synthetic dataset + 3D scatter 
import numpy as np
import torch
import matplotlib.pyplot as plt
from mpl_toolkits.mplot3d import Axes3D  # noqa: F401 (needed for 3D)

# Reproducibility
torch.manual_seed(0)
np.random.seed(0)

N = 5000
x = (torch.rand(N, 2) * 6.0) - 3.0                 # uniform in [-3, 3]^2
y = torch.cos(2.0 * x[:, 0]) + torch.cos(3.0 * x[:, 1])
y = y.unsqueeze(1)                                  # shape (N, 1)

# 3D scatter (generated when you run this)
fig = plt.figure(figsize=(6, 5))
ax = fig.add_subplot(111, projection="3d")
ax.scatter(x[:,0].numpy(), x[:,1].numpy(), y[:,0].numpy(), s=3)
ax.set_xlabel("x1"); ax.set_ylabel("x2"); ax.set_zlabel("y")
plt.show()

Model: 2 → 128 → 1 (ReLU hidden, identity output)

python • regression FFN + training loop 
import torch.nn as nn
from torch.utils.data import TensorDataset, DataLoader

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

ds = TensorDataset(x, y)
loader = DataLoader(ds, batch_size=256, shuffle=True)

model = nn.Sequential(
    nn.Linear(2, 128),
    nn.ReLU(),
    nn.Linear(128, 1),
).to(device)

criterion = nn.MSELoss()
optimizer = torch.optim.Adam(model.parameters(), lr=3e-2)

def train_epochs(epochs=50):
    model.train()
    history = []
    for ep in range(epochs):
        losses = []
        for xb, yb in loader:
            xb, yb = xb.to(device), yb.to(device)
            optimizer.zero_grad()
            pred = model(xb)
            loss = criterion(pred, yb)
            loss.backward()
            optimizer.step()
            losses.append(loss.item())
        history.append(float(np.mean(losses)))
        if (ep + 1) % 10 == 0:
            print(f"epoch {ep+1:03d} | mse {history[-1]:.6f}")
    return history

loss_hist = train_epochs(epochs=60)

Predict on a grid: meshgrid → model → reshape → surface

To draw a smooth surface, you evaluate the model on a grid of \((x_1, x_2)\) points. The key is reshaping: grids are 2D, the model wants a 2-column matrix, and the output must be reshaped back to the grid.

python • meshgrid predictions + 3D surface plot (run locally) 
@torch.no_grad()
def plot_surface(model, lim=3.0, steps=80):
    model.eval()
    xs = torch.linspace(-lim, lim, steps)
    ys = torch.linspace(-lim, lim, steps)
    X, Y = torch.meshgrid(xs, ys, indexing="ij")       # shapes (steps, steps)

    grid = torch.stack([X.reshape(-1), Y.reshape(-1)], dim=1).to(device)  # (steps*steps, 2)
    Z = model(grid).reshape(steps, steps).cpu()

    # Plot
    import matplotlib.pyplot as plt
    from mpl_toolkits.mplot3d import Axes3D  # noqa: F401
    fig = plt.figure(figsize=(7, 5))
    ax = fig.add_subplot(111, projection="3d")
    ax.plot_surface(X.numpy(), Y.numpy(), Z.numpy(), linewidth=0, antialiased=True)
    ax.set_xlabel("x1"); ax.set_ylabel("x2"); ax.set_zlabel("pred y")
    plt.show()

plot_surface(model)

What you’re checking visually: does the learned surface match the “bumpy” cosine structure? If not, try adjusting learning rate, epochs, batch size, or hidden width.

9) Hyperparameters: a practical experimentation workflow

Hyperparameters (learning rate, batch size, number of epochs, hidden width, optimizer choice) don’t come with universal rules that always win. A reliable workflow is:

  1. Start reasonable (common defaults: Adam or SGD, moderate LR, moderate batch size)
  2. Confirm learning (loss decreases; accuracy increases)
  3. Search locally (try a small grid around current settings)
  4. Use evidence (plots/metrics) to decide next changes

Minimal grid-search skeleton

python • tiny hyperparameter sweep skeleton 
def run_experiment(lr, hidden, epochs=3):
    model = nn.Sequential(
        nn.Linear(28*28, hidden),
        nn.ReLU(),
        nn.Linear(hidden, 10),
    ).to(device)

    opt = torch.optim.SGD(model.parameters(), lr=lr)
    crit = nn.CrossEntropyLoss()

    for _ in range(epochs):
        model.train()
        for x, y in train_loader:
            x = x.view(x.size(0), -1).to(device)
            y = y.to(device)
            opt.zero_grad()
            logits = model(x)
            loss = crit(logits, y)
            loss.backward()
            opt.step()

    acc = accuracy_loader(model, test_loader, device)
    return acc

for lr in [0.3, 0.1, 0.03]:
    for hidden in [64, 128, 256]:
        acc = run_experiment(lr=lr, hidden=hidden, epochs=2)
        print(f"lr={lr:<5} hidden={hidden:<4} -> test acc {acc:.4f}")

Goal: don’t hunt for mystical “best” defaults—build a habit of quick, disciplined experiments.
