Transformers & Attention

Where this fits Chapter 10 (RNNs) gave you a sequence model that processes one token at a time and keeps a hidden state. That gives you sequential dependence but poor parallelism, vanishing gradients on long context, and an information bottleneck through one hidden vector. The Transformer (Vaswani et al., 2017) replaced the recurrent step with attention — every token looks at every other token in parallel. Every modern LLM you'll fine‑tune in Chapter 12 is a Transformer.

1) The intuition: a soft, content‑based dictionary lookup

Think of attention as a dictionary lookup that is differentiable.

  • For each token you want to update, you compute a query vector q: "what kind of context do I need?"
  • For every token in the sequence (including itself) you have a key k ("what kind of context do I provide?") and a value v ("what would I contribute if attended to?").
  • You score each (q, k) pair, turn the scores into a probability distribution with softmax, and return the weighted average of values.

That's it. The whole Transformer is this operation, stacked, with a small feed‑forward network in between, and residual connections + LayerNorm.

2) Scaled dot‑product attention

Stack the queries, keys and values into matrices $Q \in \mathbb{R}^{n\times d_k}$, $K \in \mathbb{R}^{n\times d_k}$, $V \in \mathbb{R}^{n\times d_v}$ where $n$ is the sequence length. Then:

$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\left(\dfrac{QK^{\top}}{\sqrt{d_k}}\right) V$

The $\sqrt{d_k}$ scaling keeps the dot products from growing too large with dimension, which would push softmax into a near‑one‑hot regime and kill the gradients. Implemented in PyTorch:

import torch, torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    d_k = Q.size(-1)
    scores = (Q @ K.transpose(-2, -1)) / d_k**0.5      # (..., n, n)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))
    weights = F.softmax(scores, dim=-1)                # (..., n, n)
    return weights @ V                                 # (..., n, d_v)

Causal (decoder) masking

For language modeling, position $i$ may only attend to positions $\le i$. The mask is the lower triangular matrix of ones; everything above the diagonal is set to $-\infty$ before softmax.

n = Q.size(-2)
causal = torch.tril(torch.ones(n, n, device=Q.device))
out = scaled_dot_product_attention(Q, K, V, mask=causal)

3) Multi‑head attention

One attention head learns one notion of relevance. Real language has many in parallel — syntactic agreement, coreference, topic, etc. Multi‑head attention runs $h$ attention operations in parallel on linear projections of $Q, K, V$ into smaller subspaces, then concatenates and projects back:

$\mathrm{MHA}(X) = \big[\mathrm{head}_1; \dots; \mathrm{head}_h\big] W^{O}, \quad \mathrm{head}_i = \mathrm{Attention}(XW_i^{Q}, XW_i^{K}, XW_i^{V})$ 
import torch, torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0
        self.h, self.d_k = n_heads, d_model // n_heads
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x, mask=None):
        B, N, _ = x.shape
        def split(t): return t.view(B, N, self.h, self.d_k).transpose(1, 2)
        Q, K, V = split(self.W_q(x)), split(self.W_k(x)), split(self.W_v(x))
        out = F.scaled_dot_product_attention(Q, K, V, is_causal=mask is None)
        out = out.transpose(1, 2).contiguous().view(B, N, self.h * self.d_k)
        return self.W_o(out)
Use the built‑in Since PyTorch 2.0, F.scaled_dot_product_attention dispatches to fused kernels (FlashAttention‑2, memory‑efficient attention) where available. Hand‑rolling the einsum version is good for understanding, not for production.

4) Position information: sinusoidal vs RoPE

Self‑attention is permutation‑equivariant — shuffle the input tokens and you get the same shuffled output. Language is not permutation‑invariant, so we add positional information.

Sinusoidal (Vaswani et al., original)

Add a fixed sinusoidal vector to each token embedding before the first layer:

$\mathrm{PE}(p, 2i) = \sin\!\bigl(p / 10000^{2i/d}\bigr),\quad \mathrm{PE}(p, 2i+1) = \cos\!\bigl(p / 10000^{2i/d}\bigr)$

Pros: simple. Cons: weak length extrapolation; baked into embeddings.

Rotary Position Embedding — RoPE (used by LLaMA, Mistral, Qwen, Gemma)

RoPE rotates pairs of dimensions of $Q$ and $K$ by an angle proportional to the position. Because it acts on $Q$ and $K$ multiplicatively rather than additively on the embedding, the dot product $Q_m^{\top} K_n$ becomes a function of the relative position $m - n$:

$\langle\mathrm{RoPE}(q, m),\ \mathrm{RoPE}(k, n)\rangle = f(q, k, m - n)$

This makes long‑context generalization much better and is why every modern open‑weights LLM uses it. Variants like YaRN and NTK‑aware scaling stretch RoPE's effective range to extend a 4k‑trained model to 32k+ tokens at inference.

5) The full Transformer block

One block (decoder‑only flavor) is:

class Block(nn.Module):
    def __init__(self, d_model, n_heads, d_ff):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = MultiHeadAttention(d_model, n_heads)
        self.ln2 = nn.LayerNorm(d_model)
        self.ff  = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.GELU(),
            nn.Linear(d_ff, d_model),
        )

    def forward(self, x):
        x = x + self.attn(self.ln1(x))   # pre‑norm residual
        x = x + self.ff(self.ln2(x))
        return x

Stack $L$ blocks, prepend a token‑embedding + positional encoding, append a final LayerNorm and an unembedding (often tied to the input embedding), and you have a GPT‑style language model.

7) Inference: KV cache

At training time you process the whole sequence in parallel. At generation time you produce one token at a time. Naively you would recompute all of $K$ and $V$ for every prefix — quadratic per token. Instead you cache the per‑layer $K$ and $V$ tensors:

  • For token $t$, compute only the new $q_t, k_t, v_t$.
  • Append $k_t, v_t$ to the cache.
  • Attention is now $q_t$ against the cached $K, V$ — linear in $t$.

The KV cache is the main reason LLM inference is memory‑bound rather than compute‑bound, and the reason context length costs RAM (each extra token adds 2 · n_layers · n_heads · d_head · dtype_bytes per request).

8) Efficiency: FlashAttention, MQA, GQA, sliding window

  • FlashAttention (Dao et al., 2022; v2 2023; v3 2024) computes the same softmax attention but tiles the work to keep $QK^{\top}$ in on‑chip SRAM and never materializes the full $n \times n$ matrix in HBM. Result: ~2–4× speedup and much less memory.
  • Multi‑Query Attention (MQA) shares one $K$/$V$ projection across all heads — shrinks the KV cache by $h\times$. Used in PaLM.
  • Grouped‑Query Attention (GQA) is the compromise: one $K$/$V$ per group of $g$ query heads. Used in LLaMA‑2/3, Mistral, Qwen2, Gemma 2 — basically the modern default.
  • Sliding‑window attention (Mistral, Gemma) restricts each token to attend only to the last $W$ tokens, lowering the attention cost from $O(n^2)$ to $O(nW)$.
  • Mixture of Experts (MoE) (Mixtral, DeepSeek‑V3) replaces the feed‑forward block with a router that picks $k$ of $E$ experts per token — more parameters, similar FLOPs.

9) Pulling it all together

  1. Embed tokens; add positional info (sinusoidal, learned, or RoPE).
  2. Stack $L$ Transformer blocks (pre‑norm + residual + GQA + SwiGLU/GELU FFN).
  3. Final LayerNorm + tied unembedding → logits over the vocabulary.
  4. For generation: KV cache + sampling (greedy / temperature / top‑p / top‑k).
