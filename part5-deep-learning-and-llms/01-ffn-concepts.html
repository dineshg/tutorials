1) Start with “Why?” and “How?”

The motivation in the lecture is simple: the brain is a physical system of neurons that exchange signals. If a neuron can be approximated as a computation, then connecting many of them (in a simulation) might produce useful behavior—what we call artificial intelligence.

Key idea: neural networks are powerful because they can learn hierarchies of features: simple patterns in early layers (edges), more meaningful parts in middle layers (eyes/noses), and full concepts later (faces). This “stacking” of representation is why the field is called deep learning.

If you want an intuition playground, the lecture mentions TensorFlow Playground: https://playground.tensorflow.org (try changing layers/units and watch non-linear boundaries appear).

2) From logistic regression to a “neuron”

A single artificial neuron is basically a linear model plus a nonlinearity. For an input vector \(x \in \mathbb{R}^d\):

\[ z = w^\top x + b \] \[ a = \sigma(z) \]

Here \(w\) are weights, \(b\) is a bias, \(z\) is the pre-activation, and \(a\) is the activated output. In binary classification, \(\sigma\) (sigmoid) turns \(z\) into a value in \((0,1)\), interpretable as a probability.

3) Feed-forward networks: architecture & forward propagation

A feed-forward neural network arranges neurons in layers, with connections flowing from input → hidden layers → output. (If connections loop back to earlier layers, that’s typically a recurrent design; the transcript contrasts these.)

Vectorized layer equation

For layer \(\ell\), we typically use:

\[ z^{(\ell)} = W^{(\ell)} a^{(\ell-1)} + b^{(\ell)} \] \[ a^{(\ell)} = g^{(\ell)}\!\left(z^{(\ell)}\right) \]

This is the same idea as the single neuron, but generalized: \(W^{(\ell)}\) is a matrix of weights, \(b^{(\ell)}\) is a bias vector, and \(g^{(\ell)}\) is an activation function applied elementwise (except for some output activations like softmax).

What changes vs. linear/logistic regression? Nothing fundamental—just multiple linear transforms chained with nonlinearities.
Why does that help? Without nonlinearities, stacked linear layers collapse into a single linear layer.
How to read it? Think “feature → feature → feature → … → final prediction”.

A 2-layer example (hidden layer + output layer)

\[ a^{(1)} = g\!\left(W^{(1)}x + b^{(1)}\right) \] \[ \hat{y} = \text{(output activation)}\!\left(W^{(2)}a^{(1)} + b^{(2)}\right) \]

4) Activation functions (why they matter)

The lecture emphasizes: activations are what give neural networks the ability to learn non-linear decision boundaries. Without them, you get only linear separators.

Sigmoid

\[ \sigma(z)=\frac{1}{1+e^{-z}} \]

Good for binary outputs, but can be problematic in deep networks due to small derivatives.

Tanh (hyperbolic tangent)

\[ \tanh(z)=\frac{e^z-e^{-z}}{e^z+e^{-z}} \]

Similar “S-shape” to sigmoid, but centered around 0 (outputs in \((-1,1)\)), which often helps internal scaling.

ReLU (Rectified Linear Unit)

\[ \mathrm{ReLU}(z)=\max(0,z) \]

The lecture’s storyline: researchers moved from “beautiful smooth” activations (sigmoid/tanh) toward ReLU because it trains deep networks more reliably in practice.

Pragmatic ML takeaway (from the transcript): experimentation beats philosophy. If you have a computer, run experiments instead of guessing outcomes in your head.

5) The vanishing gradient problem (and why ReLU helped)

When training with backpropagation, gradients for earlier layers are products of many derivatives (chain rule). If these derivatives are often small, multiplying them repeatedly can make the gradient shrink toward zero for early layers—so those layers barely learn.

Sigmoid derivative is small most of the time

\[ \sigma'(z)=\sigma(z)\bigl(1-\sigma(z)\bigr) \]

The derivative peaks at \(0.25\). Repeated multiplication like \((0.25)^5 \approx 0.0009766\) shrinks fast.

A related visualization used in discussions of vanishing gradients

Historically (as mentioned in the lecture), one workaround was layer-wise pretraining (e.g., Hinton’s earlier-era approaches). Modern practice usually relies on better activations (ReLU-family), initialization, normalization, and optimizers.

6) Multiclass classification: Softmax + Cross-Entropy

For \(K\) classes, the output layer produces a vector of logits \(z \in \mathbb{R}^K\). We convert logits into a probability distribution with softmax:

\[ \mathrm{softmax}(z)_i = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}} \]

This guarantees probabilities are positive and sum to 1. In practice, we usually do not apply softmax explicitly during training; we use a numerically-stable loss that combines softmax with cross-entropy.

Three common tasks ↔ output activations

  • Regression: identity output (no activation): \(\hat{y}=z\)
  • Binary classification: sigmoid output: \(\hat{y}=\sigma(z)\)
  • Multiclass classification: softmax output: \(\hat{y}=\mathrm{softmax}(z)\)

The lecture also notes: softmax can handle binary classification too (with \(K=2\)), but it’s redundant because the two probabilities always sum to 1.

7) Images as tensors (and why we “flatten” for a basic FFN)

Computers store images as arrays. A color image (RGB) is typically a tensor of shape:

\[ \text{image} \in \mathbb{R}^{H \times W \times C}, \quad C=3 \]

Each pixel channel is often quantized into 256 levels (0–255). For neural networks, it’s common to scale to \([0,1]\).

Grayscale is simpler

MNIST digits are grayscale, so each image is \(28 \times 28\) (no RGB channels).

Flattening

A fully-connected FFN expects vectors. Flattening turns a \(28\times 28\) image into a vector of length \(784\):

\[ x \in \mathbb{R}^{28\times 28} \;\;\longrightarrow\;\; \mathrm{vec}(x)\in \mathbb{R}^{784} \]

8) MNIST feed-forward network in PyTorch (with mini-batches)

The lecture’s example: MNIST has 10 classes (digits 0–9), so it’s multiclass classification. A simple FFN can look like: 784 → 128 → 10, with ReLU in the hidden layer.

Model equations (as described)

\[ a^{(1)} = \mathrm{ReLU}(W^{(1)}x + b^{(1)}) \] \[ z^{(2)} = W^{(2)}a^{(1)} + b^{(2)} \] \[ \hat{y} = \mathrm{softmax}(z^{(2)}) \quad \text{(conceptually; training uses cross-entropy)} \]

Mini-batch gradient descent: instead of computing gradients over the entire dataset at once, we compute them on small batches. PyTorch’s DataLoader handles batching for you.

PyTorch code (end-to-end)

This follows the transcript’s steps: load MNIST, flatten + scale, build FFN, train with batches, evaluate, and inspect mistakes. 

import torch
import torch.nn as nn
import torch.nn.functional as F
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# 1) Data: MNIST (grayscale 28x28)
# - ToTensor() scales uint8 [0,255] -> float [0,1]
# - We'll flatten in the training loop (or use transforms.Lambda)
transform = transforms.ToTensor()

train_ds = datasets.MNIST(root="data", train=True, download=True, transform=transform)
test_ds  = datasets.MNIST(root="data", train=False, download=True, transform=transform)

train_loader = DataLoader(train_ds, batch_size=128, shuffle=True, num_workers=2, pin_memory=True)
test_loader  = DataLoader(test_ds, batch_size=256, shuffle=False, num_workers=2, pin_memory=True)

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")


# 2) Model: 784 -> 128 -> 10
class FFN(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(28*28, 128)
        self.fc2 = nn.Linear(128, 10)

    def forward(self, x):
        # x: (B, 1, 28, 28) -> flatten to (B, 784)
        x = x.view(x.size(0), -1)
        x = F.relu(self.fc1(x))
        logits = self.fc2(x)  # raw scores (logits)
        return logits

model = FFN().to(device)

# 3) Training setup
# CrossEntropyLoss = LogSoftmax + NLLLoss (numerically stable)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

def accuracy_from_logits(logits, y):
    preds = logits.argmax(dim=1)
    return (preds == y).float().mean().item()

# 4) Train loop (mini-batch gradient descent)
def train_one_epoch():
    model.train()
    total_loss = 0.0
    total_acc = 0.0
    n_batches = 0

    for x, y in train_loader:
        x, y = x.to(device), y.to(device)

        optimizer.zero_grad()
        logits = model(x)
        loss = criterion(logits, y)
        loss.backward()
        optimizer.step()

        total_loss += loss.item()
        total_acc += accuracy_from_logits(logits, y)
        n_batches += 1

    return total_loss / n_batches, total_acc / n_batches

@torch.no_grad()
def evaluate():
    model.eval()
    total_loss = 0.0
    total_acc = 0.0
    n_batches = 0

    for x, y in test_loader:
        x, y = x.to(device), y.to(device)
        logits = model(x)
        loss = criterion(logits, y)

        total_loss += loss.item()
        total_acc += accuracy_from_logits(logits, y)
        n_batches += 1

    return total_loss / n_batches, total_acc / n_batches

# 5) Run training
for epoch in range(1, 6):
    tr_loss, tr_acc = train_one_epoch()
    te_loss, te_acc = evaluate()
    print(f"epoch {epoch:02d} | train loss {tr_loss:.4f} acc {tr_acc:.4f} | test loss {te_loss:.4f} acc {te_acc:.4f}")


# 6) Inspect mistakes (what the model gets wrong)
@torch.no_grad()
def collect_mistakes(max_items=24):
    model.eval()
    mistakes = []
    for x, y in test_loader:
        x, y = x.to(device), y.to(device)
        logits = model(x)
        preds = logits.argmax(dim=1)
        wrong_mask = preds != y
        if wrong_mask.any():
            xw = x[wrong_mask].cpu()
            yw = y[wrong_mask].cpu()
            pw = preds[wrong_mask].cpu()
            for i in range(xw.size(0)):
                mistakes.append((xw[i], int(yw[i]), int(pw[i])))
                if len(mistakes) >= max_items:
                    return mistakes
    return mistakes

mistakes = collect_mistakes()
print("Collected mistakes:", len(mistakes))
