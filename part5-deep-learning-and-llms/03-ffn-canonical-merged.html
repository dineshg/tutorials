Why? How? Why this topic is so interesting

The phrase artificial neural network comes from a big idea: biological brains are made of neurons that communicate using electrical and chemical signals. Once you accept that a brain is a physical information-processing system, a natural question appears:

The motivating question

If a brain is “a kind of computer”, can we simulate networks of neurons on a computer to create artificial intelligence?

Of course this is a simplification—real brains are messy: many kinds of neurons, many regions, and connections that crisscross in complex ways. But for a computational model, we start with a deliberately simplified neuron, connect many of them, and see what the system can learn.

Two network “wiring” patterns
  • Feedforward: information flows layer by layer from input to output.
  • Recurrent / feedback: connections go from a later layer back to an earlier layer (a “loop”).

This tutorial focuses on feedforward networks (also called “MLPs”).

From logistic regression to a “neuron”

The lecture connects a single logistic regression model to a simplified neuron: the model computes a weighted sum, adds a bias (threshold), then passes it through a squashing nonlinearity (sigmoid) and finally makes a binary decision.

Single neuron (binary classification)
$$a = \mathbf{w}^\top \mathbf{x} + b \quad\text{(activation / logit)}$$ $$\hat{p} = \sigma(a) = \frac{1}{1 + e^{-a}} \quad\text{(probability of class 1)}$$ $$\hat{y} = \mathbb{1}[\hat{p} \ge 0.5] \quad\text{(predict 1 if prob ≥ 0.5)}$$
Interpretation

$\mathbf{w}$ controls how strongly each input feature influences the decision. $b$ shifts the decision threshold.

In biology, neurons sum incoming signals through synapses. Some synapses are “strong” (large magnitude weight), others “weak” (small weight). Some are excitatory (positive weight), others inhibitory (negative weight). A neuron either “fires” or does not (an all-or-nothing spike), which resembles a binary decision.

From one neuron to layers to deep networks

One neuron can only compute one feature. The next step is to use many neurons in parallel. A whole layer can learn many different features from the same input.

One hidden layer (vectorized)
$$\mathbf{a}^{(1)} = \mathbf{W}^{(1)}\mathbf{x} + \mathbf{b}^{(1)}$$ $$\mathbf{z}^{(1)} = \phi\!\left(\mathbf{a}^{(1)}\right) \quad\text{(apply activation element-wise)}$$ $$\mathbf{a}^{(2)} = \mathbf{W}^{(2)}\mathbf{z}^{(1)} + \mathbf{b}^{(2)}$$

Notation note: the lecture uses superscripts $(1),(2),\dots,(L)$ for layer index, and often uses $L$ for the total number of layers.

Why matrices?

With multiple neurons per layer, writing sums like $w_1x_1 + \dots + w_Dx_D$ becomes unreadable. Matrices let us compute all neurons’ weighted sums efficiently in one operation.

A helpful perspective: “a sequence of feature transformations”

A modern way to think about a deep network: each layer transforms the representation (features) into a new representation. Early layers learn simple patterns; later layers can combine them into more complex ones. This “feature hierarchy” is part of why the field is called deep learning.

Classic example (as discussed)

In face recognition visualizations: early layers resemble edge detectors, mid layers detect parts (eyes, nose, mouth), and later layers represent entire faces.

Neural nets as geometry solvers: why linear models aren’t enough

The lecture repeatedly emphasizes: machine learning is a geometry problem. A linear classifier draws a hyperplane; points on different sides get different labels. But many real problems require curved decision boundaries.

Two ways to make models “more powerful”
  • Add more input features (feature engineering).
  • Add nonlinearity + layers (neural networks) to create curved boundaries automatically.

Deep learning reduces the need for manual feature engineering—especially for unstructured data like images, audio, and text.

If you remove activation functions, a deep “network” collapses into a linear map (a key lecture point): composing linear functions yields another linear function. Nonlinear activations are what make deep nets able to model more complex geometry.

Activation functions: sigmoid → tanh → ReLU (and why)

Sigmoid

Sigmoid maps real numbers to $(0,1)$. It is natural for binary outputs interpreted as probabilities. But hidden layers want their activations to be well-behaved for learning.

Tanh (hyperbolic tangent)

Tanh is “sigmoid-like” but centered around $0$, producing outputs in $(-1,1)$. This better matches the idea of keeping layer inputs roughly centered (the lecture connects this to standardization).

Key formulas
$$\sigma(x) = \frac{1}{1+e^{-x}}$$ $$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$$

Exercise idea from the lecture: show $\tanh$ is a scaled/shifted version of sigmoid.

ReLU and modern activations

Sigmoid/tanh are smooth, but deep networks historically struggled with them because of vanishing gradients. ReLU is not perfectly smooth at $0$, but empirically trains deep nets much better.

ReLU
$$\mathrm{ReLU}(x) = \max(0, x)$$

The lecture mentions common variants: Leaky ReLU (small slope for $x<0$), ELU, and Softplus. In practice, many people still use ReLU as a strong default.

How models learn: gradients, chain rule, backprop, and vanishing gradients

Training is still “minimize a loss with gradient descent,” but neural networks are compositions of many functions. Backpropagation is essentially the chain rule applied efficiently.

Gradient descent (high level)

Parameter update
$$\theta \leftarrow \theta - \eta \nabla_\theta \mathcal{L}$$

$\eta$ is the learning rate (a hyperparameter). Too large: divergence. Too small: painfully slow.

Chain rule (the key math for backprop)

If $y = f(g(x))$, then $$\frac{dy}{dx} = \frac{dy}{dg}\cdot \frac{dg}{dx}.$$ For deeper compositions $f_1(f_2(\dots f_L(x)\dots))$ you multiply many derivatives. This multiplication is exactly what can cause gradients to shrink (vanish).

Vanishing gradients (as described)

For sigmoid, the derivative is at most $0.25$. Multiplying numbers ≤ $0.25$ many times makes the result tiny, so early layers receive almost no learning signal.

Sigmoid derivative and the “0.25” fact
$$\sigma'(x) = \sigma(x)\left(1-\sigma(x)\right) \le 0.25$$ $$0.25^5 \approx 0.00098,\qquad 0.1^5 = 0.00001$$

The lecture also mentions an older workaround: layer-wise pretraining (associated with Geoffrey Hinton and students), where layers were trained one at a time. Modern practice usually avoids this by using better activations (like ReLU) and other techniques.

PyTorch connection (important practical note)

You typically do not compute derivatives by hand. PyTorch uses automatic differentiation. Your job is to build the computation graph and call loss.backward().

Three tasks & their final-layer activations

A key summary from the lecture: regardless of whether the “body” of your network is a linear model, MLP, CNN, or RNN, the final layer follows the task type.

Regression: identity (no activation) Binary classification: sigmoid Multiclass classification: softmax
Final layer patterns

(Let $\mathbf{a}$ be the final layer’s logits.)

$$\textbf{Regression:}\quad \hat{y} = a$$ $$\textbf{Binary:}\quad \hat{p} = \sigma(a)$$ $$\textbf{Multiclass:}\quad \hat{\mathbf{p}} = \mathrm{softmax}(\mathbf{a})$$
“Softmax is more general” (lecture note)

Softmax can handle $K$ classes including $K=2$, but for binary classification sigmoid is simpler and avoids redundancy.

Multiclass classification: softmax + cross-entropy (and stability)

For $K$ classes, the final layer outputs a vector of logits $\mathbf{a}\in\mathbb{R}^K$. We want probabilities: each is nonnegative and they sum to 1. Softmax does exactly that:

Softmax
$$\mathrm{softmax}(\mathbf{a})_k = \frac{e^{a_k}}{\sum_{j=1}^{K} e^{a_j}}.$$

Numerator is always positive. Denominator sums all numerators, so probabilities sum to 1.

Training uses cross-entropy loss. A crucial numerical stability tip (mirroring the BCE-with-logits story): you usually don’t apply softmax “manually” before the loss. Frameworks combine these steps to avoid overflow/underflow.

PyTorch rule of thumb

Use nn.CrossEntropyLoss with raw logits (no softmax layer). It internally computes a stable log-softmax + negative log-likelihood.

Cross-entropy (one-hot target)
$$\mathcal{L} = -\sum_{k=1}^{K} y_k \log(\hat{p}_k).$$

For prediction, you do not need probabilities either: the class with maximum logit is the same class with maximum softmax probability. So prediction is:

Prediction
$$\hat{y}=\arg\max_k a_k.$$

How images are represented in a computer (RGB, grayscale, quantization, flattening)

The lecture transitions from “model” topics to “data” topics: deep learning shines on unstructured data like images, sound, and text. But how does an image become an input vector?

RGB: three numbers per pixel

A color image has height and width, but each pixel also has color. A common representation is RGB (Red, Green, Blue). Each pixel stores how much red, green, and blue it contains.

So a color image is naturally a 3D tensor: $$A[i,j,k],\quad k\in\{R,G,B\}.$$ A dataset of $N$ images becomes a 4D tensor: $$A[n,i,j,k].$$

Quantization (0 to 255)

Pixel intensities are often stored as integers in $\{0,\dots,255\}$ (8 bits per channel). That’s $256^3 \approx 16.8$ million possible RGB colors.

A quick size estimate from the lecture: a raw $500\times 500$ RGB image stores $500\cdot 500\cdot 3 = 750{,}000$ bytes $\approx 732$ KB. Compression (JPEG/PNG) reduces this significantly.

Hex colors (web dev connection)

Hex colors like #RRGGBB are just the same 8-bit values written in base-16.

Grayscale images

If an image has no color channels, you only store one intensity per pixel. Then an image is a 2D array (height × width). MNIST digits are grayscale, so each image is $28\times 28$.

Scaling to $[0,1]$ for neural networks

The lecture notes a common preprocessing step: convert uint8 pixel values in $[0,255]$ to floating point values in $[0,1]$. This keeps inputs small and numerically convenient.

Flattening: the “dumb but simple” trick

A fully-connected network expects a 2D array of shape $(N, D)$. The lecture’s simple solution: flatten each $28\times 28$ image into a length-$784$ vector (scan pixels left-to-right, top-to-bottom).

Important practical note

Flattening ignores spatial structure. Convolutional neural networks (CNNs) usually perform better for vision, but flattening is a great first step to learn the training pipeline.

Mini example: a $3\times 3$ grayscale image

Think of the image as a matrix $I\in\mathbb{R}^{3\times 3}$. Flattening (also called vectorization) stacks pixels into a vector $x\in\mathbb{R}^{9}$.

Image matrix $I$ (each entry is one pixel intensity)
$I_{11}$$I_{12}$$I_{13}$
$I_{21}$$I_{22}$$I_{23}$
$I_{31}$$I_{32}$$I_{33}$

A common convention (used in the lecture) is row-major order: $$x = \mathrm{vec}(I) = [I_{11}, I_{12}, I_{13}, I_{21}, I_{22}, I_{23}, I_{31}, I_{32}, I_{33}]^\top.$$ In general, for a $H\times W$ grayscale image: $$x_{(i-1)W + j} = I_{ij},\quad 1\le i\le H,\; 1\le j\le W.$$

For MNIST: $28\times 28 \to 784$

MNIST is grayscale, so $C=1$. Each sample arrives as a tensor of shape $[B, 28, 28]$ (or sometimes $[B,1,28,28]$), and we reshape it into $[B, 784]$ before sending it into a fully-connected layer. 

# x: [B, 28, 28]
x = x.view(x.size(0), -1)   # [B, 784]  (the “dumb but simple” trick)
This is not “feature engineering”—it’s only changing the shape of the data so a dense layer can consume it.

MNIST tutorial: multiclass ANN with DataLoader, mini-batches, GPU

MNIST is a classic dataset of handwritten digits (0–9). This is a multiclass classification problem with $K=10$ classes. The training set has $60{,}000$ images and the test set has $10{,}000$ images.

Why mini-batches?

Large datasets may not fit in memory. So we compute loss/gradients on small batches. A batch is a random sample—its gradient is a noisy but useful estimate of the full gradient.

PyTorch code: end-to-end (clean baseline)

This code matches the lecture flow: load MNIST from torchvision, scale to $[0,1]$ via ToTensor(), flatten to 784 features, build a small feedforward net, train with mini-batches, evaluate with accuracy and confusion matrix. 

import torch
import torch.nn as nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# -------------------------
# 1) Data
# -------------------------
transform = transforms.ToTensor()  # converts uint8 [0..255] to float32 [0..1]

train_ds = datasets.MNIST(root=".", train=True, transform=transform, download=True)
test_ds  = datasets.MNIST(root=".", train=False, transform=transform, download=True)

batch_size = 128
train_loader = DataLoader(train_ds, batch_size=batch_size, shuffle=True)
test_loader  = DataLoader(test_ds, batch_size=batch_size, shuffle=False)

# -------------------------
# 2) Model (flatten -> FC -> ReLU -> FC)
# -------------------------
model = nn.Sequential(
    nn.Linear(28 * 28, 128),
    nn.ReLU(),
    nn.Linear(128, 10)  # logits for 10 classes
)

# -------------------------
# 3) Device (GPU if available)
# -------------------------
device = torch.device("cuda:0" if torch.cuda.is_available() else "cpu")
model.to(device)

# -------------------------
# 4) Loss + Optimizer
# -------------------------
criterion = nn.CrossEntropyLoss()  # expects logits, applies stable log-softmax internally
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# -------------------------
# 5) Train / Eval helpers
# -------------------------
def accuracy_from_logits(logits, y):
    # logits: [B, 10], y: [B]
    preds = torch.argmax(logits, dim=1)
    return (preds == y).float().mean().item()

def run_epoch(loader, training: bool):
    total_loss = 0.0
    total_correct = 0
    total_seen = 0

    if training:
        model.train()
    else:
        model.eval()

    for x, y in loader:
        x = x.to(device)          # [B, 1, 28, 28]
        y = y.to(device)          # [B]

        x = x.view(x.size(0), -1) # flatten -> [B, 784]

        if training:
            optimizer.zero_grad()

        with torch.set_grad_enabled(training):
            logits = model(x)               # [B, 10]
            loss = criterion(logits, y)

            if training:
                loss.backward()
                optimizer.step()

        total_loss += loss.item() * x.size(0)
        total_correct += (torch.argmax(logits, dim=1) == y).sum().item()
        total_seen += x.size(0)

    avg_loss = total_loss / total_seen
    acc = total_correct / total_seen
    return avg_loss, acc

# -------------------------
# 6) Training loop
# -------------------------
epochs = 10
for ep in range(1, epochs + 1):
    train_loss, train_acc = run_epoch(train_loader, training=True)
    test_loss,  test_acc  = run_epoch(test_loader, training=False)
    print(f"Epoch {ep:02d} | train loss {train_loss:.4f} acc {train_acc:.4f} | "
          f"test loss {test_loss:.4f} acc {test_acc:.4f}")

Confusion matrix and “what does the model confuse?”

The lecture suggests going beyond accuracy: find which digits are confused. A confusion matrix counts how often each true label is predicted as each label. (You often expect large values on the diagonal.) 

import numpy as np
from sklearn.metrics import confusion_matrix

model.eval()
all_preds = []
all_true = []

with torch.no_grad():
    for x, y in test_loader:
        x = x.to(device).view(x.size(0), -1)
        logits = model(x)
        preds = torch.argmax(logits, dim=1).cpu().numpy()
        all_preds.append(preds)
        all_true.append(y.numpy())

y_pred = np.concatenate(all_preds)
y_true = np.concatenate(all_true)

cm = confusion_matrix(y_true, y_pred)
print(cm)  # rows=true labels, cols=pred labels
Lecture-style interpretation idea

If the matrix shows “4 is often predicted as 9”, look at example images: some handwritten 4s really do resemble 9s.

Also mentioned: the dataset pipeline uses transform=ToTensor(). That’s why printing raw train_ds.data shows integers 0–255, but batches from the DataLoader are floats in 0–1.

Synthetic regression example: why visualization matters (conceptually)

The lecture argues that synthetic datasets are valuable because you can see where a model succeeds or fails. For a 2D input $(x_1, x_2)$ you can even visualize the target function as a 3D surface.

Note about plots

This page avoids generating custom plots (per your request). The code below shows the standard approach, but we do not embed a plot image unless we have a reliable reference image for that exact function. 

import numpy as np
import torch
import torch.nn as nn

# Synthetic data: x in [-3, 3]^2, y = cos(2*x1) + cos(3*x2)
N = 10_000
X = (np.random.rand(N, 2) * 6.0) - 3.0
y = np.cos(2 * X[:, 0]) + np.cos(3 * X[:, 1])

X_t = torch.tensor(X, dtype=torch.float32)
y_t = torch.tensor(y, dtype=torch.float32).view(-1, 1)

model = nn.Sequential(
    nn.Linear(2, 128),
    nn.ReLU(),
    nn.Linear(128, 1)  # regression: no activation
)

opt = torch.optim.Adam(model.parameters(), lr=3e-2)
loss_fn = nn.MSELoss()

for ep in range(200):
    opt.zero_grad()
    preds = model(X_t)
    loss = loss_fn(preds, y_t)
    loss.backward()
    opt.step()

Hyperparameters: “experimentation, not philosophy”

The transcript ends with an important mindset: there is no universal formula for choosing learning rates, number of layers, hidden units, activations, optimizers, etc. These are hyperparameters. You discover what works by testing and observing.

Practical strategy (matches the lecture)
  • Start with common defaults (Adam, ReLU, learning rates like $10^{-3}$).
  • Run experiments; plot loss/accuracy curves; watch for divergence or slow learning.
  • Use literature in your domain as a guide for reasonable ranges.
  • Remember: rules of thumb change; results are empirical.

The “don’t predict the outcome with your mind—run the program” theme is a reminder that deep learning is an experimental science. When you can test an idea quickly, testing beats guessing.
