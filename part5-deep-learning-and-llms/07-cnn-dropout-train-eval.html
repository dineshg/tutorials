Dropout: what it actually does (and what it does NOT do)

Correct mental model: Dropout applies a new random mask every forward pass (typically every mini-batch). It does not permanently “turn off” a neuron, and it does not remove it for the whole training run.

What gets dropped?

In standard dropout, we drop activations (the outputs of a layer), not the parameter tensors directly. Think: “for this forward pass, some activation values become zero.”

Training mode math (inverted dropout)

Let a layer output be \(h\). Dropout samples a mask \(m\) where: \[ m_i \sim \mathrm{Bernoulli}(q), \quad q = 1-p \] Then it computes: \[ \tilde{h} = \frac{m \odot h}{q} \]

  • If \(m_i = 0\): that activation is zeroed for this pass.
  • If \(m_i = 1\): it’s kept, and scaled by \(1/q\).
  • The scaling keeps the expected activation magnitude consistent between training and inference.

Inference mode math

In eval mode, dropout is disabled: \[ \tilde{h} = h \] No randomness, no masking.

What happens to gradients when an activation is dropped?

When an activation is dropped (\(m_i=0\)), it contributes nothing to the output, so the gradient through that activation is zero for that sample: \[ \frac{\partial \tilde{h}_i}{\partial h_i} = \frac{m_i}{q} \] If \(m_i=0\), then \(\partial \tilde{h}_i / \partial h_i = 0\). Practically:

  • Weights feeding into that activation receive no update from that sample.
  • But next iteration, the mask changes, so the same unit may be active and updated then.

PyTorch: the exact behavior you should remember

  • nn.Dropout(p) applies random masking only when the module is in train() mode.
  • In eval() mode, it becomes an identity function (no change).
  • Masks are typically sampled per forward call (practically per mini-batch).
python • verify mask randomness in train vs determinism in eval 
import torch
import torch.nn as nn

drop = nn.Dropout(p=0.5)
x = torch.ones(1, 10)

drop.train()
y1 = drop(x)
y2 = drop(x)  # different mask, usually different output

drop.eval()
y3 = drop(x)
y4 = drop(x)  # same output, dropout disabled

print("train different:", not torch.allclose(y1, y2))
print("eval same:", torch.allclose(y3, y4))

Where to put dropout (practical default)

Most commonly: between dense layers. For conv feature maps, many projects skip it or use specialized variants.

python • common pattern: Dense → ReLU → Dropout → Dense 
import torch.nn as nn

head = nn.Sequential(
    nn.Linear(512, 256),
    nn.ReLU(),
    nn.Dropout(p=0.5),
    nn.Linear(256, 10)  # logits
)

For conv nets, you may also see nn.Dropout2d / nn.Dropout3d: these drop entire channels (feature maps), not individual pixels.

train() vs eval(): why dropout “turns off” for predictions

PyTorch modules can have different behavior depending on whether you’re training or evaluating. Dropout is the classic example:

  • model.train() → dropout ON (random masking), batchnorm uses batch stats
  • model.eval() → dropout OFF (no masking), batchnorm uses running stats

Standard pattern: call model.train() before the training loop, and model.eval() during validation/testing (usually with torch.no_grad()).

python • training/evaluation skeleton (dropout-safe) 
model.train()
for xb, yb in train_loader:
    optimizer.zero_grad()
    logits = model(xb)        # dropout active here
    loss = loss_fn(logits, yb)
    loss.backward()
    optimizer.step()

model.eval()
correct = 0
total = 0
with torch.no_grad():
    for xb, yb in val_loader:
        logits = model(xb)    # dropout disabled here
        preds = logits.argmax(dim=1)
        correct += (preds == yb).sum().item()
        total += yb.size(0)
print("val acc:", correct / total)

Dropout FAQ (common confusions)

“Does dropout stop training a random node each iteration?”

It temporarily zeros a random subset of activations each forward pass. Those dropped activations have zero gradient for that sample, so their incoming weights don’t update from that sample. Next iteration, the mask changes, so they can update again. Not permanent

“Is the same set of nodes masked all the time?”

No—standard dropout resamples a new mask per forward pass. Some specialized variants (e.g., “locked/variational dropout” in some sequence models) reuse a mask across time steps, but that’s not the default behavior of nn.Dropout.

“Does dropout remove weights?”

No. The parameters are still there. Dropout modifies the activations during training. At inference, nothing is removed.

“Why is there scaling by 1/(1-p)?”

Without scaling, the average activation magnitude would shrink during training. Inverted dropout keeps the expected activation scale similar between train and eval.

“Should I use dropout in conv layers?”

Often not necessary. If you do, consider channel-wise dropout (Dropout2d) rather than pixel-wise dropout, and verify by experiment—sometimes it helps, sometimes it’s neutral.
