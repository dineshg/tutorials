Part II — Backend, Platform & Security

The platform layer that the agent and ML systems sit on top of: FastAPI basics, the request lifecycle, Pydantic data contracts, concurrency & user isolation, background work and retries, the full authentication stack (SSO/OAuth2/OIDC/PKCE/JWT vs opaque), workload identity vs end‑user identity, multi‑tenant isolation, external integrations, and the GitHub governance that keeps the codebase shippable.

Reading order tip Chapters 1–6 are the FastAPI runtime and data-contract story. Chapter 4 is the Pydantic bridge between raw HTTP/JSON and typed Python objects. Chapters 7–11 are the auth story, ordered from OIDC basics through modern sender-constrained patterns. Chapters 12–14 then merge the identity concepts with tenancy, IAM, and GCP workload identity. Chapters 15–16 cover external integrations and GitHub governance.
Prerequisites Comfortable with Python 3, basic HTTP, and the command line. No prior FastAPI knowledge required. The auth chapters (6–15) are theory + code and stand on their own; you do not need to read Part I first.

Identity path Chapters 12 and 13 intentionally cover related identity ideas from different angles: Chapter 12 separates end-user identity, workload identity, and IAM; Chapter 13 applies the same model to tenant isolation, AD groups, and bootstrap policy. Read them together before the GCP implementation chapter.
