What you’ll build (and the math behind it)
  • A linear classifier with logit $a = w^\top x + b$ and probability $p = \sigma(a)$.
  • A training objective (binary cross-entropy) and a training loop that minimizes it: $$\theta^* = \arg\min_{\theta} \frac{1}{n}\sum_{i=1}^{n} \ell\bigl(a_i(\theta), y_i\bigr).$$

Notation used throughout: $x\in\mathbb{R}^D$ (features), $y\in\{0,1\}$ (label), $w\in\mathbb{R}^D$ (weights), $b\in\mathbb{R}$ (bias).

1) The ML Workflow: What Happens Inside fitpredictscore?

A typical scikit-learn-style workflow follows the same outer structure for many models:

  1. Load data into inputs \(X\) and targets \(y\).
  2. Instantiate a model (e.g., logistic regression, linear regression, etc.).
  3. Train it with something like model.fit(X, y).
  4. Make predictions with model.predict(X) (on train or on new/test data).
  5. Evaluate with model.score(X, y).
Why this question matters

On the surface, “linear regression” and “logistic regression” can look identical in code: you mostly change the class name. The difference is what the model computes, what loss it minimizes, and how training updates parameters. The inside of these functions is where the real meaning is.

In math terms, fit is “choose parameters to minimize a loss”, predict is “run the model forward”, and score is “compute a metric”. For classification, the most common metric is accuracy.

2) Classification Accuracy and Error

In binary classification (e.g., spam vs not-spam), every prediction is either correct or incorrect.

$$\text{accuracy}=\frac{\#\text{correct}}{\#\text{total}}, \qquad \text{error}=\frac{\#\text{wrong}}{\#\text{total}}=1-\text{accuracy}.$$

Equivalently, using an indicator function $\mathbf{1}[\cdot]$: $$\text{accuracy} = \frac{1}{n}\sum_{i=1}^{n}\mathbf{1}[\hat{y}_i = y_i].$$

Train vs test accuracy

You can compute accuracy on the training set and on a held-out test set: train accuracy measures how well you fit the seen data, while test accuracy estimates how well you generalize to unseen data.

Note: if classes are highly imbalanced (e.g., 99% of samples are class 0), accuracy can be misleading. It’s still a good starting point, but you may later want precision/recall or ROC-AUC.

3) Geometry: Linear Decision Boundary and the Logit

A linear classifier draws a boundary in feature space. For a 2D input \(x=(x_1,x_2)\), the boundary is a line:

$$w_1x_1+w_2x_2+b=0.$$

This form generalizes cleanly to higher dimensions (where the boundary is a hyperplane). It also connects directly to the key scalar quantity the model computes:

$$a=w^\top x + b,$$

where \(a\) is commonly called the activation or logit.

Signed distance intuition

The value $a = w^\top x + b$ tells you which side of the boundary you’re on. If you normalize by $\lVert w\rVert$, then $\frac{a}{\lVert w\rVert}$ is proportional to the signed distance from the point to the hyperplane.

Why not just use \(y=mx+b\)?

In classification, the axes are features \(x_1,x_2,\dots\). The label \(y\) is not an axis. The vector form \(w^\top x+b\) works for any number of features \(D\), not just 2.

From geometry to prediction (sign test)

If a point is not on the boundary, then \(a=w^\top x+b\) will be either positive or negative. One side of the hyperplane always yields \(a>0\), the other yields \(a<0\).

$$\hat{y}=\begin{cases} 1 & \text{if } a>0 \\ 0 & \text{if } a\le 0 \end{cases}$$

4) Step Function vs Sigmoid: Logistic Regression

The sign test is like a hard “step function.” In deep learning we prefer smooth differentiable functions, so we often map the logit \(a\) through a sigmoid:

$$\sigma(a)=\frac{1}{1+e^{-a}} \in (0,1).$$

We interpret \(p=\sigma(a)\) as a probability:

$$p(y=1\mid x)=\sigma(w^\top x + b).$$

A useful identity: the log-odds are linear in $x$: $$\log\frac{p}{1-p} = w^\top x + b.$$ This is one reason logistic regression is so interpretable.

And convert to a class prediction by thresholding:

$$\hat{y}=\mathbf{1}[p\ge 0.5].$$

Terminology you’ll see
  • Logistic function = sigmoid function.
  • Logit = the sigmoid input \(a=w^\top x+b\).
  • Logistic regression = linear model + sigmoid for binary classification.

5) Training: Loss Functions, Gradients, and Learning Rate

Regression vs classification: why the loss changes

In regression, targets \(y\in\mathbb{R}\), predictions \(\hat{y}\in\mathbb{R}\), so losses like mean squared error (MSE) make sense:

$$\text{MSE}=\frac{1}{n}\sum_{i=1}^{n}(y_i-\hat{y}_i)^2.$$

In binary classification, the model outputs probabilities, and a standard loss is binary cross-entropy:

$$\mathcal{L}_{\text{BCE}} = -\frac{1}{n}\sum_{i=1}^{n}\left[y_i\log(p_i) + (1-y_i)\log(1-p_i)\right],\quad p_i=\sigma(a_i).$$

Where BCE comes from (one sentence)

If you assume $y\mid x$ follows a Bernoulli distribution with probability $p=\sigma(w^\top x+b)$, then minimizing BCE is exactly minimizing the negative log-likelihood.

Loss / cost / objective

The lecture used these words interchangeably: they all mean “the scalar we minimize during training.”

How does a model learn?

PyTorch uses automatic differentiation to compute gradients of the loss with respect to parameters \(w\) and \(b\). Gradient descent updates parameters iteratively:

$$w \leftarrow w - \eta \frac{\partial \mathcal{L}}{\partial w},\qquad b \leftarrow b - \eta \frac{\partial \mathcal{L}}{\partial b},$$

where \(\eta\) is the learning rate.

Learning rate is a hyperparameter

It is not learned by the model; you choose it (often by trial + intuition). A common starting sweep is \(10^{-1},10^{-2},10^{-3},\dots\).

6) Dataset: Breast Cancer (scikit-learn) + Preprocessing

The lecture uses the well-known breast cancer dataset included in scikit-learn. Loading it is a single function call (no manual download needed):

  • Inputs: data.data \(\rightarrow X\)
  • Targets: data.target \(\rightarrow y\) (0/1)
  • Names: data.target_names, data.feature_names
Two preprocessing steps from the lecture
  1. Train/test split so we can evaluate on unseen data.
  2. Standardization with StandardScaler: fit on train only, then transform train and test using the train mean/std.

Standardization is: $$x' = \frac{x-\mu}{\sigma}$$ computed per feature (dimension). It helps optimization because different features can start on very different scales.

7) PyTorch Implementation: Logistic Regression End-to-End

We will implement the lecture’s PyTorch pipeline: load data, split, scale, build model, train with gradient descent, then evaluate accuracy.

Python Load + split + scale 
import numpy as np
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

# 1) Load data (built into sklearn)
data = load_breast_cancer()
X = data.data               # shape (n, D)
y = data.target             # shape (n,), values in {0,1}

# (Optional inspection)
print("X shape:", X.shape)  # e.g., (569, 30)
print("y shape:", y.shape)
print("target names:", data.target_names)  # e.g., ['malignant', 'benign']

# 2) Train/test split (test_size ~ 33% as in the lecture)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.33, random_state=123, stratify=y
)

# 3) Standardize features
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)   # fit only on train
X_test  = scaler.transform(X_test)        # transform test using train stats

n, D = X_train.shape
print("train:", X_train.shape, "test:", X_test.shape)

Model: Linear layer + sigmoid

The lecture builds a logistic regression model as:

$$p=\sigma(w^\top x+b),$$

which in PyTorch can be represented as nn.Linear(D,1) then nn.Sigmoid(), wrapped in nn.Sequential.

Shape sanity check

For binary classification, it’s common to keep outputs as shape $(n,1)$. That’s why the targets are reshaped with .view(-1, 1).

Python Model + loss + optimizer + tensor conversion 
import torch
import torch.nn as nn

# Logistic regression = Linear + Sigmoid
model = nn.Sequential(
    nn.Linear(D, 1),
    nn.Sigmoid()
)

# Binary cross entropy loss (for probabilities)
criterion = nn.BCELoss()

# Adam optimizer (common default in deep learning)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# Convert numpy -> torch tensors (float32)
X_train_t = torch.from_numpy(X_train.astype(np.float32))
X_test_t  = torch.from_numpy(X_test.astype(np.float32))

# IMPORTANT in PyTorch: y should be shape (n, 1) for this setup
y_train_t = torch.from_numpy(y_train.astype(np.float32)).view(-1, 1)
y_test_t  = torch.from_numpy(y_test.astype(np.float32)).view(-1, 1)

Training loop (gradient descent with autograd)

The lecture’s loop is the standard PyTorch pattern: zero gradients → forward pass → loss → backward → optimizer step.

Python Train + track train/test loss (per epoch) 
train_losses = []
test_losses = []

epochs = 300

for epoch in range(epochs):
    # ---- TRAIN step ----
    optimizer.zero_grad()
    p_train = model(X_train_t)                 # probabilities in (0,1)
    loss_train = criterion(p_train, y_train_t)
    loss_train.backward()
    optimizer.step()

    # ---- EVAL losses (no gradient needed) ----
    with torch.no_grad():
        p_test = model(X_test_t)
        loss_test = criterion(p_test, y_test_t)

    train_losses.append(loss_train.item())
    test_losses.append(loss_test.item())

    if (epoch+1) % 50 == 0:
        print(f"epoch {epoch+1:4d}  train_loss={loss_train.item():.4f}  test_loss={loss_test.item():.4f}")
Why “test loss increasing” matters

If training loss keeps decreasing but test/validation loss starts increasing, you may be overfitting (memorizing train patterns that don’t generalize).

Accuracy computation (and the “mean of booleans” trick)

The model outputs probabilities. For predictions, round to 0/1:

$$\hat{y}=\text{round}(p).$$

Then accuracy is the mean of element-wise equality: in Python, True behaves like 1 and False like 0.

Python Train/test accuracy 
with torch.no_grad():
    p_train = model(X_train_t).numpy()
    p_test  = model(X_test_t).numpy()

# probabilities -> class predictions
yhat_train = np.round(p_train)
yhat_test  = np.round(p_test)

train_acc = (yhat_train.reshape(-1) == y_train).mean()
test_acc  = (yhat_test.reshape(-1)  == y_test).mean()

print("train accuracy:", train_acc)
print("test  accuracy:", test_acc)

8) A Better Logistic Regression in PyTorch: BCEWithLogitsLoss

The lecture points out a practical numerical-stability issue: explicitly computing sigmoid uses exponentials, and BCE uses logs. PyTorch provides a stable combined loss that works directly on logits.

Conceptually, it’s still the same loss, just computed in a numerically stable way. For a single logit $a$ and label $y\in\{0,1\}$, one stable form is: $$\ell(a,y)=\max(a,0) - a\,y + \log\left(1+e^{-|a|}\right).$$

Instead of computing \(p=\sigma(a)\) then BCE, we keep the model output as:

$$a=w^\top x+b\quad\text{(logit)},$$

and use nn.BCEWithLogitsLoss.

Python Stable version: logits + BCEWithLogitsLoss 
# Model now outputs logits (no Sigmoid layer!)
model = nn.Linear(D, 1)

# Stable combined loss
criterion = nn.BCEWithLogitsLoss()

optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# Training loop is the same shape, but now model(X) returns logits
for epoch in range(epochs):
    optimizer.zero_grad()
    logits_train = model(X_train_t)                 # any real number
    loss_train = criterion(logits_train, y_train_t)
    loss_train.backward()
    optimizer.step()

# Predictions: logits > 0  <=>  sigmoid(logits) > 0.5
with torch.no_grad():
    logits_test = model(X_test_t).numpy()
yhat_test = (logits_test > 0).astype(np.int64).reshape(-1)
Prediction rule without sigmoid

Because \(\sigma(a)\ge 0.5 \iff a\ge 0\), you can classify directly using the sign of the logit: \(a>0 \Rightarrow \hat{y}=1\), else \(\hat{y}=0\).

9) Saving and Loading Models (and Downloading from Colab)

The key object: state_dict

In PyTorch, a model’s parameters live in an ordered dictionary returned by:

$$\texttt{model.state\_dict()} \Rightarrow \{\text{weights},\text{biases},\dots\}.$$

Python Save + reload (recommended pattern) 
# Save
torch.save(model.state_dict(), "mymodel.pt")

# Recreate the same architecture (must match!)
model2 = nn.Sequential(
    nn.Linear(D, 1),
    nn.Sigmoid()
)

# Load weights
state = torch.load("mymodel.pt", map_location="cpu")
model2.load_state_dict(state)
model2.eval()

# Verify identical accuracy (example)
with torch.no_grad():
    p_test2 = model2(X_test_t).numpy()
yhat_test2 = np.round(p_test2).reshape(-1)
print("test accuracy (reloaded):", (yhat_test2 == y_test).mean())

10) Why Logistic Regression is a “Neuron”

A single neuron-like computation in deep learning is:

$$a=\sum_{i=1}^{D} w_i x_i + b,\qquad \hat{y}=\sigma(a)\ \text{(then threshold)}.$$

So logistic regression is literally a single neuron: a weighted sum + bias, followed by a nonlinear activation. Deep networks are just many of these stacked together.

The lecture’s biological analogy:

  • Inputs \(x_i\) correspond to signals from many incoming connections (dendrites).
  • Weights \(w_i\) represent connection strengths (strong vs weak).
  • The sign of \(w_i\) represents excitatory (positive) vs inhibitory (negative) influence.
  • A threshold-like effect resembles the “all-or-nothing” firing behavior.
Interpretation of weights (kept from lecture)

If \(w_i\approx 0\), feature \(x_i\) has little effect on the output. If \(|w_i|\) is large, small changes in \(x_i\) strongly affect the output. The sign determines direction of influence.

11) Train / Validation / Test: Why Splits Exist (and How Kaggle Changes the Words)

Why split at all?

The whole point of a classifier is to work on future unseen data. If you only measure performance on past seen data, you can fool yourself.

Overfitting intuition (from lecture)

Complex models can fit training data extremely well but generalize poorly. We want a model that balances being expressive enough to learn patterns, but not so expressive that it memorizes noise.

Train vs validation vs test: the naming confusion

If you only split into two sets, the “test” set is effectively acting as a validation set (you are using it to validate generalization). That’s why people sometimes say “train/validation” and sometimes “train/test” for the same two-way split.

Kaggle contest semantics (from the lecture)

  • Kaggle provides a labeled “train” set and an unlabeled “test” set.
  • You can’t compute test accuracy yourself (labels are hidden).
  • So you split Kaggle’s train into train + validation to tune choices.
  • Public leaderboard vs private leaderboard: private is used for final ranking to reduce overfitting to the public test feedback.

Cross-validation (kept, with clean wording)

A common classical ML approach is \(k\)-fold cross-validation: split data into \(k\) parts, train on \(k-1\), validate on the remaining part, repeat \(k\) times, then summarize validation performance. The lecture notes it’s less common in deep learning due to large datasets and long training times.

What is validation used for?

Validation performance is how you choose hyperparameters: learning rate, optimizer type, feature subsets, model architecture, number of layers/units, activation choices, etc. You can’t just “gradient descent” on architecture, so you use trial-and-error guided by validation scores.
