Chapter 2 • Linear Regression

Linear Regression and Training in PyTorch

We’ll connect the math of linear regression to a working PyTorch training loop, keeping the link between equations and code as clear as possible.

In the previous chapter we viewed machine learning as a geometry problem: data become points in a space; models are shapes (lines, planes, curves) that we fit or use to separate those points. In this chapter we make that concrete with the simplest possible model: linear regression in PyTorch.

2.1 A Simple Dataset and Notation

Consider a dataset with one input feature and one target. For example:

  • $x$: years of experience
  • $y$: salary

We have $N$ samples:

$$ (x_1, y_1), (x_2, y_2), \dots, (x_N, y_N). $$

Here, capital $N$ denotes the total number of samples. The input values $x_i$ and target values $y_i$ are fixed; our job is to learn a function that maps $x_i \mapsto \hat{y}_i$ so that predictions $\hat{y}_i$ are close to the true values $y_i$.

In PyTorch, we store the data as tensors:

import torch

# Toy example
x = torch.tensor([1.0, 2.0, 3.0, 4.0]).view(-1, 1)  # shape: (N, 1)
y = torch.tensor([2.0, 4.0, 6.0, 8.0]).view(-1, 1)  # shape: (N, 1)

The .view(-1, 1) reshapes the 1D tensors into column vectors of shape $(N, 1)$, which is the common shape for a single input feature and single output in PyTorch.

2.2 The Model: A Straight Line

For simple linear regression, our model is a straight line:

$$ \hat{y} = m x + b. $$

For each sample $i$:

$$ \hat{y}_i = m x_i + b. $$

The parameters $m$ (slope) and $b$ (intercept) are unknown; these are what we want to learn from data.

In PyTorch we represent this line with nn.Linear:

import torch.nn as nn

# One input feature, one output
model = nn.Linear(in_features=1, out_features=1)

This model internally stores a weight and a bias, which correspond to $m$ and $b$: 

print(model.weight)  # corresponds roughly to m
print(model.bias)    # corresponds roughly to b

To get predictions, we call the model like a function:

y_pred = model(x)  # computes y_hat = m * x + b for all samples

Conceptually, this is computing all the $\hat{y}_i = m x_i + b$ at once using tensor operations.

2.3 The Loss Function: Mean Squared Error

To evaluate how good a particular line is, we measure how far the predictions are from the true values. For regression, the standard choice is Mean Squared Error (MSE):

$$ L(m, b) = \frac{1}{N} \sum_{i=1}^{N} \bigl(y_i - \hat{y}_i\bigr)^2. $$

Substituting the model $\hat{y}_i = m x_i + b$, we can write:

$$ L(m, b) = \frac{1}{N} \sum_{i=1}^{N} \bigl(y_i - (m x_i + b)\bigr)^2. $$

Geometrically, $\lvert y_i - \hat{y}_i \rvert$ is the vertical distance from the point $(x_i, y_i)$ to the line. Squaring and averaging gives us a single non-negative number that is small when the line fits the data well and large when it does not.

In PyTorch, we use nn.MSELoss:

criterion = nn.MSELoss()

y_pred = model(x)
loss = criterion(y_pred, y)  # scalar tensor: the MSE
print(loss.item())

At this point we have everything needed to define a supervised learning problem:

  • a model $\hat{y} = m x + b$, and
  • a loss function $L(m, b)$ that tells us how good any particular choice of $m$ and $b$ is.

2.4 Learning as Minimizing the Loss

The data $(x_i, y_i)$ are given and fixed. The variables in the loss expression are the parameters $m$ and $b$. Learning means:

$$ (m^*, b^*) = \arg\min_{m, b} L(m, b). $$

In other words, we want to find the values of $m$ and $b$ that make the mean squared error as small as possible.

For very simple models like this, you can derive explicit formulas for $m^*$ and $b^*$ by taking derivatives of $L$ with respect to $m$ and $b$, setting them to zero, and solving:

$$ \frac{\partial L}{\partial m} = 0, \qquad \frac{\partial L}{\partial b} = 0. $$

However, this approach does not generalize to more complex models, such as deep neural networks, where the loss depends on thousands or millions of parameters in highly nonlinear ways. Instead, we use gradient-based iterative methods.

For a parameter vector $\theta$ (in our case $\theta = {m, b}$), classic gradient descent updates parameters as:

$$ \theta \leftarrow \theta - \eta \nabla_\theta L(\theta), $$

where $\eta > 0$ is the learning rate. Applied to our parameters, this becomes:

$$ m \leftarrow m - \eta \frac{\partial L}{\partial m}, \qquad b \leftarrow b - \eta \frac{\partial L}{\partial b}. $$

PyTorch computes these derivatives automatically using automatic differentiation. Our job is to define the model, the loss, and the optimization procedure; PyTorch will handle the gradients.

2.5 The Optimizer: SGD

We now need a component that takes the gradients and updates the parameters. PyTorch provides several optimizers; the simplest is Stochastic Gradient Descent (SGD): 

import torch.optim as optim

learning_rate = 0.01
optimizer = optim.SGD(model.parameters(), lr=learning_rate)

Here, model.parameters() returns an iterator over the model’s trainable parameters (in this case, the weight and bias). The optimizer will update these in-place, using the gradients that PyTorch computes.

2.6 The Training Loop

Training consists of repeating the same sequence of steps: zeroing gradients, computing predictions, computing the loss, computing gradients, and updating parameters. Each full pass through these steps is one training iteration; when we apply this to the whole dataset repeatedly, we often call each pass through the dataset an epoch.

A typical training loop for simple linear regression looks like this:

num_epochs = 1000

            for epoch in range(num_epochs):
                    # 1. Zero out previous gradients
                    optimizer.zero_grad()

                    # 2. Forward pass: compute predictions
                    y_pred = model(x)

                    # 3. Compute the loss
                    loss = criterion(y_pred, y)

                    # 4. Backward pass: compute gradients dL/dm and dL/db
                    loss.backward()

                    # 5. Update parameters using the optimizer
                    optimizer.step()

                    # (Optional) monitor progress
                    if (epoch + 1) % 100 == 0:
                            print(f"Epoch {epoch + 1}, Loss = {loss.item():.4f}")

The mapping to the math is direct:

  • y_pred = model(x) computes all $\hat{y}_i = m x_i + b$.
  • loss = criterion(y_pred, y) computes $L(m, b)$.
  • loss.backward() fills in model.weight.grad and model.bias.grad with $\frac{\partial L}{\partial m}$ and $\frac{\partial L}{\partial b}$.
  • optimizer.step() performs the gradient descent updates.

2.7 Getting Results Out

After training, we may want to inspect predictions or plot the learned line. Since many plotting libraries expect NumPy arrays, we often convert our tensors: 

with torch.no_grad():       # disable gradient tracking
        y_pred = model(x)       # final predictions

x_np = x.detach().numpy()
y_np = y.detach().numpy()
y_pred_np = y_pred.detach().numpy()

The detach() call removes tensors from the computation graph, making conversion to NumPy safe. We can then plot x_np versus y_np and y_pred_np using Matplotlib, or use them in further analysis.

2.8 Summary

In this chapter we connected the geometric picture of linear regression to both its mathematical formulation and its PyTorch implementation. We started with a set of data points $(x_i, y_i)$, defined a linear model:

$$ \hat{y} = m x + b, $$

and measured its performance using the mean squared error:

$$ L(m, b) = \frac{1}{N} \sum_{i=1}^{N} \bigl(y_i - (m x_i + b)\bigr)^2. $$

We then viewed learning as minimizing this loss with respect to the parameters $m$ and $b$:

$$ (m^*, b^*) = \arg\min_{m, b} L(m, b), $$

and we implemented this using PyTorch’s tensors, nn.Linear model, nn.MSELoss loss function, and an SGD optimizer inside a training loop.

This simple example is more than a toy: the same pattern—define data, define model, define loss, define optimizer, write a training loop—will reappear in every deep learning model we build with PyTorch, from basic neural networks to convolutional and recurrent architectures.
