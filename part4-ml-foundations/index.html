Part IV — ML Foundations

The geometric, code‑first foundations every later chapter depends on. We start with “what is a model, geometrically?”, then implement linear regression and linear classification end‑to‑end in PyTorch, then cover the tree/ensemble family (Random Forest, AdaBoost, Gradient Boosting, XGBoost) — the family that still beats deep learning on tabular data.

Prerequisites Python 3 fluency, basic linear algebra (vectors, matrix multiply, derivatives). No prior PyTorch needed — Chapter 1 is the on‑ramp. Read this part before Part V (Deep Learning & LLMs).
What you can do after this part Train and evaluate a regression or classification model in PyTorch from scratch, and ship a tabular ML baseline using XGBoost. You can then proceed to Part V with no gaps.
ML foundations spine Chapter 2 is the conceptual owner: data points, model shapes, regression, classification, and supervised learning. Chapters 3 and 4 intentionally repeat the same fit/predict/score workflow because regression and classification differ in target type, output activation, loss, and metrics. Chapter 5 then branches to tree and ensemble methods as the tabular baseline family that often competes with or beats neural networks on structured data.

Chapters in order

  1. Introduction to PyTorch
  2. Machine learning as geometry
  3. Linear regression and the training loop
  4. Linear classification (logistic regression)
  5. Trees, bagging & boosting (Random Forest, AdaBoost, XGBoost)
