1) What is an Ensemble? Big Picture

Ensemble learning means we combine many simple models to create one strong model. The individual models are called base learners (often decision trees).

Why does combining models help?

  • Reduce variance: if one model is “noisy”, averaging many models smooths errors.
  • Reduce bias: if one model is “too simple”, sequential correction can capture complex patterns.
  • Stability: ensembles are less sensitive to small changes in data.

The two main families

Bagging builds many models independently and averages/votes.

Boosting builds models sequentially, each correcting earlier mistakes.

Key theme: Bagging focuses on variance reduction. Boosting focuses on bias reduction (and can also reduce variance with regularization).

2) Decision Trees: How They Learn Foundation

What a split really is

At a node, a decision tree searches for a question like: $\text{feature}_j \le \text{threshold}$ that divides the data into two groups (left/right).

The tree wants the children nodes to be more “pure” than the parent — meaning each child contains mostly one class (for classification), or reduced spread (for regression).

The tree’s “greedy” rule

Trees usually use a greedy strategy: pick the best split right now, then repeat inside each child node.

Greedy means: it doesn’t look ahead many steps; it chooses the best local improvement. That’s why trees are fast and intuitive — but also why a single tree can be unstable.
Optional: what does “best” mean in practice?

“Best” means the split that most reduces a chosen impurity measure (Gini/Entropy), or for modern boosting (XGBoost) the split that most reduces the training loss (objective). The spirit is identical: choose the split that creates the most useful separation.

3) Impurity Measures: Gini and Information Gain Splitting Criteria

Impurity is a number that tells you how mixed the classes are inside a node. Lower impurity = node is closer to “all one class”.

Gini Impurity (CART; very common)

Idea: if a node is mostly one class, it’s pure; if it’s evenly mixed, it’s impure.

$$\mathrm{Gini}(\text{node}) = 1 - \sum_k p_k^2$$
  • Binary example: if p = 0.7 and 0.3 → Gini = 1 − (0.49 + 0.09) = 0.42
  • Pure node: p = 1.0 and 0.0 → Gini = 0

Entropy & Information Gain (ID3/C4.5 style)

Idea: entropy measures “surprise” or uncertainty in the class label.

$$\mathrm{Entropy}(\text{node}) = -\sum_k p_k\,\log_2(p_k)$$

A split is chosen by maximizing Information Gain:

$$IG = \mathrm{Entropy}(\text{parent}) - \left(\frac{N_L}{N}\,\mathrm{Entropy}(L) + \frac{N_R}{N}\,\mathrm{Entropy}(R)\right)$$
How the tree uses impurity:
For each candidate split, compute the children impurities, take a weighted average, then choose the split with the largest impurity reduction.
Gini reduction formula (same shape as Information Gain)
$$\mathrm{GiniSplit} = \frac{N_L}{N}\,\mathrm{Gini}(L) + \frac{N_R}{N}\,\mathrm{Gini}(R) \qquad \mathrm{GiniGain} = \mathrm{Gini}(\text{parent}) - \mathrm{GiniSplit}$$

4) A Concrete Split Example (Step-by-Step) Intuition

Imagine a node contains 10 training samples for binary classification (Yes/No): 6 Yes, 4 No.

Parent impurity (Gini)

p(Yes)=0.6, p(No)=0.4

$$\mathrm{Gini}(\text{parent}) = 1 - (0.6^2 + 0.4^2) = 1 - (0.36 + 0.16) = 0.48$$

Try a candidate split

Suppose we test: $\text{Age} \le 30$

  • Left child: 6 samples → 5 Yes, 1 No
  • Right child: 4 samples → 1 Yes, 3 No

Child impurities

$$\begin{aligned} \mathrm{Gini}(L) &= 1 - \left(\left(\frac{5}{6}\right)^2 + \left(\frac{1}{6}\right)^2\right) = 1 - \frac{26}{36} = \frac{10}{36} \approx 0.2778 \\ \mathrm{Gini}(R) &= 1 - \left(\left(\frac{1}{4}\right)^2 + \left(\frac{3}{4}\right)^2\right) = 1 - \frac{10}{16} = \frac{6}{16} = 0.375 \end{aligned}$$

Weighted average & gain

$$\begin{aligned} \mathrm{GiniSplit} &= \frac{6}{10}\cdot 0.2778 + \frac{4}{10}\cdot 0.375 = 0.3167 \\ \mathrm{GiniGain} &= 0.48 - 0.3167 = 0.1633 \end{aligned}$$

The tree would compare this gain with all other possible splits and choose the biggest.

Core takeaway: every split is “just” computing impurity before and after. Ensembles (bagging/boosting) change how trees are trained, not the basic logic of splitting.

5) Bagging and Random Forests Variance Reduction

Bagging in plain words

Bagging trains many trees on slightly different versions of the dataset. Each tree makes its own prediction, then we combine predictions by averaging (regression) or voting (classification).

Bagging is powerful because averaging many noisy models tends to cancel out random errors.

How do we get “different versions” of the data?

Using bootstrap sampling: sample N points from the training set with replacement. Some points repeat, some are missing.

  • Each tree sees a different dataset → trees become less correlated.
  • Less correlation → averaging works better.

Random Forest: Bagging + Feature Randomness

Random Forest adds one more trick: at each split, it considers only a random subset of features.

  • This makes trees even more diverse.
  • Diversity improves the power of averaging.

What split criterion is used?

Usually Gini impurity for classification (CART-style), and variance/MSE for regression.

In Random Forest, trees are independent, so you can train trees in parallel very easily.
Common Random Forest terms
  • n_estimators: number of trees.
  • max_features: how many features to try at each split.
  • OOB (out-of-bag) error: estimate accuracy using samples left out of each bootstrap.

6) Boosting Overview Bias Reduction

Boosting trains trees one after another. Each new tree is trained to correct mistakes made by the current ensemble.

Boosting as “learning from errors”

Suppose the first model is a little better than random. It will still make mistakes on “hard” examples. Boosting increases focus on those hard examples, so the next model improves where it matters.

Over many rounds, the ensemble becomes very strong.

Two major boosting styles

  • AdaBoost: reweights training examples; uses a weighted vote.
  • Gradient Boosting / XGBoost: minimizes a loss function using gradients; adds trees as corrections.
Boosting is sequential at the “tree level”: tree (t) depends on results from tree (t−1).

7) AdaBoost (Adaptive Boosting) — Deep Dive Weights + Weak Learners

AdaBoost is easiest to understand if you imagine the algorithm is “turning up the volume” on points it got wrong. Those points become more important in the next round.

What AdaBoost uses as base learners

Most commonly: decision stumps (depth-1 trees). A stump is one split like: $\text{feature}_j \le \text{threshold}$.

A stump is weak by itself, but many stumps combined can form a complex boundary.

AdaBoost is typically described for binary labels: $y \in \{-1, +1\}$. (It can be extended, but binary is the cleanest story.)

AdaBoost algorithm (step-by-step)

Initialize weights: w_i = 1/N for i=1..N

For t = 1..T:
  1) Train weak learner h_t(x) using weights w_i
  2) Compute weighted error:
       ε_t = Σ w_i * 1[y_i ≠ h_t(x_i)]
  3) Compute learner weight (importance):
       α_t = 0.5 * ln( (1 − ε_t) / ε_t )
  4) Update sample weights:
       w_i ← w_i * exp( −α_t * y_i * h_t(x_i) )
     (misclassified points get upweighted)
  5) Normalize weights so Σ w_i = 1

Final prediction:
  H(x) = sign( Σ α_t * h_t(x) )

What does α mean?

αt is how much we trust the stump. If a stump has low error, it gets a larger α (more voting power).

$$\alpha_t = \tfrac{1}{2}\ln\left(\frac{1-\varepsilon_t}{\varepsilon_t}\right)$$

Why weights focus on mistakes

If a point is misclassified, the term $\exp(-\alpha\, y\, h(x))$ becomes exp(+α), increasing its weight. Correct points shrink by exp(−α).

This is why AdaBoost can be sensitive to noise/outliers: if an outlier is repeatedly misclassified, its weight can become huge.

How splitting/impurity works inside AdaBoost

AdaBoost changes the “importance” of each training sample via weights. That means when the stump chooses a split, it effectively uses a weighted impurity (or weighted classification error).
Weighted Gini idea (intuition)

In a normal node, class probability is “count of class / total count”. In AdaBoost, it becomes “sum of weights of class / sum of all weights”.

$$p_k = \frac{\sum_i w_i\,\mathbf{1}[y_i = k]}{\sum_i w_i} \qquad \mathrm{WeightedGini} = 1 - \sum_k p_k^2$$

The stump still tries to reduce impurity — but now mistakes have larger weights, so splits that fix mistakes become more attractive.

Mini numeric example (why a weak learner can get high α)

If ε = 0.2 (20% weighted error), then:

$$\alpha = \tfrac{1}{2}\ln\left(\frac{0.8}{0.2}\right) = \tfrac{1}{2}\ln(4) \approx 0.693$$

That’s a strong positive weight, meaning the stump gets a meaningful vote. If ε approaches 0.5, α approaches 0 (weak vote).

8) Gradient Boosting and XGBoost Loss Optimization

Gradient Boosting generalizes boosting by saying: “Let’s minimize a loss function (objective) by adding trees that move us in the best direction.”

Gradient Boosting idea in simple steps

  1. Start with a simple prediction (like the average).
  2. Compute how wrong you are (residuals / gradients).
  3. Train a small tree to predict those errors.
  4. Add the tree (scaled by a learning rate) to improve predictions.
  5. Repeat.

Additive model view:

$$\hat{y}^{(t)}(x) = \hat{y}^{(t-1)}(x) + \eta\, f_t(x)$$

Here η is the learning rate (shrinkage). Smaller η usually needs more trees but improves generalization.

What makes XGBoost special?

Second-order optimization

Uses gradients (g) and Hessians (h) to choose better splits.

More accurate split scoring

Regularization

Penalizes complex trees to reduce overfitting. (Think: don’t let the model become “too wiggly”.)

Better generalization

Engineering for speed

Histogram splits, cache efficiency, parallel computation, GPU support, distributed training.

Much faster on large data

How does XGBoost decide splits (vs Gini/Entropy)?

Random Forest uses impurity reduction (Gini/Entropy).
XGBoost uses objective reduction (loss + regularization).

But conceptually, both are choosing the split that gives the “best improvement”.
Split “gain” intuition (no heavy math)

For a candidate split, XGBoost summarizes the node using two numbers: the sum of gradients (G) and sum of Hessians (H) in that node. It can estimate how much the objective improves if we split into left and right.

$$\mathrm{Gain} \approx \mathrm{Score}(L) + \mathrm{Score}(R) - \mathrm{Score}(\text{parent}) - \gamma$$

Where γ is a penalty for creating a new split (discourages unnecessary splits).

Parallel-friendly split finding (histograms)

XGBoost often uses a histogram algorithm:

  • Bucket each feature into bins (like 256 bins).
  • Accumulate gradient/Hessian sums per bin.
  • Scan bins to find the best threshold.

This makes split-finding fast and parallelizable across features and bins.

9) Sequential vs Parallel in Boosting (XGBoost) Important Detail

Boosting is sequential across trees because tree t needs the gradients/residuals from the current model (trees 1..t−1). So you can’t train all trees at once.

So how can XGBoost still be fast?

Because XGBoost parallelizes within each boosting round. Think: “We can’t write Chapter 5 before Chapter 4, but many people can work in parallel on Chapter 4.”

What can be parallelized inside a round?

  • Gradient & Hessian computation per data point (independent operations).
  • Split search across features (each feature can be evaluated separately).
  • Histogram building (bin counts/aggregations can be parallelized).
  • Node processing at the same depth (level-wise growth enables parallel node expansion).

Distributed / multi-GPU idea

  • Each worker computes local gradient statistics.
  • Workers do a reduce (sum) to combine stats.
  • Then the best split is chosen globally.
Trees stay sequential, but the expensive computations inside each tree are distributed.
What XGBoost does NOT parallelize
  • Training tree (t+1) before finishing tree (t)
  • Skipping gradient dependency (that would break boosting)

10) Random Forest vs XGBoost Practical Comparison

Aspect Random Forest (Bagging) XGBoost (Boosting)
Training style Independent trees (parallel) Sequential trees (each corrects errors)
Main benefit Reduces variance, very robust Reduces bias strongly, often best accuracy
Split criterion Gini / Entropy (classification) or MSE (regression) Objective gain using gradients & Hessians + regularization
Noise sensitivity Usually more tolerant More sensitive; needs regularization/tuning
Hyperparameters Fewer critical knobs More knobs; more control (and more tuning)
Parallelism Across trees Within each tree / round (features, bins, nodes)
Rule of thumb: If you want a strong baseline with little tuning, Random Forest is great. If you want maximum performance on tabular data and can tune/regularize, XGBoost often wins.

11) Quick Cheat Sheet Remember Fast

Bagging (Random Forest)

  • Goal: reduce variance.
  • How: bootstrap samples + random feature subsets.
  • Combine: vote / average.
  • Splits: Gini/Entropy (classification).
  • Parallel: yes, across trees.

Boosting (AdaBoost / XGBoost)

  • Goal: reduce bias by fixing errors.
  • AdaBoost: reweight samples; weighted vote with α.
  • Gradient Boosting: fit residuals/gradients; add trees.
  • XGBoost: optimized objective + regularization + fast splits.
  • Parallel: not across trees, but inside each round.
If you only remember one thing:
Trees split by reducing “badness” (impurity or loss).
Bagging improves stability by averaging many independent trees.
Boosting improves accuracy by sequentially correcting mistakes.
