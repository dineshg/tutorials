Machine Learning as Geometry

A concrete mental model: data become points, and learning becomes fitting (or separating) shapes in space.

Most descriptions of machine learning make it sound mysterious: computers “learn from experience” and “improve automatically.” In this book, we’ll use a much simpler, concrete view:

Machine learning is, at its core, a geometry problem.

We turn data into points in a space, then learn shapes—lines, planes, and curves—that fit or separate those points.

1. Data as Points, Models as Shapes

Every machine learning problem has two parts:

  1. Features → points in space
    • A person becomes a point with coordinates like (years of experience, age, education level, …).
    • A patient becomes a point like (height, weight, blood pressure, …).
    • Each example is a point in a 2D, 3D, or higher-dimensional space.
  2. Model → a geometric object
    • In 2D: a line or curve
    • In 3D: a plane or surface
    • In higher dimensions: a hyperplane or more complex surface

Learning means adjusting that geometric object so that it matches the data well.

2. Regression: Fitting Numbers

Regression is about predicting a number: a salary, a house price, a temperature.

Example: Salary vs Experience

We want to predict salary from years of experience:

  • For each person, we record
    • \( x \) = years of experience
    • \( y \) = salary
  • Each person is a point \((x, y)\) on a 2D graph.

We look for a line:

\[ y = mx + b \]

  • \( m \): how much salary increases per year
  • \( b \): salary when experience is zero

The learned \( m \) and \( b \) define our model: given a new \( x \), we predict \( y \).

With more features (experience, age, education, location), each person becomes a point in higher dimensions, and the model becomes a plane or hyperplane instead of a line. With nonlinear relationships, we replace straight lines with curves or more flexible surfaces—but the geometric idea is the same: we fit a shape to a cloud of points.

3. Classification: Separating Categories

Classification is about predicting a category: spam/not spam, dog/cat, fraud/not fraud, at risk/not at risk.

Example: Cardiovascular Risk from Height and Weight

We want to predict cardiovascular risk from:

  • \( x_1 \) = height
  • \( x_2 \) = weight

We encode the label as:

  • 0 = not at risk
  • 1 = at risk

Steps:

  1. Plot each patient as a point \((x_1, x_2)\) in 2D.
  2. Color points by label (e.g. blue = 0, green = 1).
  3. Find a line that separates the colors as well as possible.

This line is a decision boundary:

  • One side → predicted “not at risk”
  • Other side → predicted “at risk”

In problems like fraud detection, each transaction is a point in a high-dimensional space. The model learns a boundary—no longer a simple visible line, but still a geometric surface—that separates “fraud” from “not fraud”.

4. Supervised Learning in One Picture

Both regression and classification are forms of supervised learning:

  • We know the inputs (features).
  • We know the targets (numbers or labels).
  • We learn a function that maps inputs to targets from many examples.

Geometrically:

  • Regression → fit a line/plane/curve through points.
  • Classification → find a boundary that separates differently labeled points.

As we move to deep learning and more complex architectures, the spaces and shapes become higher-dimensional and more flexible. But the fundamental picture remains:

Data are points.
Models are shapes.
Learning is finding the shapes that best match the points.
